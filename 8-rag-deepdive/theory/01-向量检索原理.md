# 向量检索原理

## 文本嵌入（Embedding）

### 什么是 Embedding？

Embedding 是将文本转换为高维向量（数字列表）的过程。语义相近的文本，其向量在空间中的距离也更近。

```
"猫喜欢吃鱼"   → [0.12, -0.34, 0.56, ..., 0.78]   ─┐
"小猫爱吃鱼"   → [0.13, -0.33, 0.55, ..., 0.77]   ─┘ 距离近（语义相似）

"今天天气不错" → [0.91, 0.22, -0.44, ..., 0.11]     ← 距离远（语义不同）
```

### 常用 Embedding 模型

| 模型 | 维度 | 来源 | 特点 |
|------|------|------|------|
| nomic-embed-text | 768 | Ollama | 本地运行，免费 |
| text-embedding-3-small | 1536 | OpenAI | 性价比高 |
| text-embedding-3-large | 3072 | OpenAI | 精度最高 |
| bge-large-zh | 1024 | BAAI | 中文优化 |
| m3e-base | 768 | Moka AI | 中文优化，开源 |

### 使用 Ollama 生成 Embedding

```python
import ollama

result = ollama.embed(
    model="nomic-embed-text",
    input="RAG 是检索增强生成技术",
)
vector = result["embeddings"][0]
print(f"向量维度: {len(vector)}")  # 768
print(f"前5个值: {vector[:5]}")
```

## 相似度计算

### 余弦相似度（Cosine Similarity）

衡量两个向量方向的一致性，值域为 `[-1, 1]`，值越大越相似。

```
            A · B
cos(θ) = ─────────
          |A| × |B|
```

```python
import numpy as np

def cosine_similarity(a, b):
    a, b = np.array(a), np.array(b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
```

### 欧几里得距离（Euclidean Distance）

衡量两个向量之间的直线距离，值越小越相似。

```
d(A, B) = sqrt(sum((a_i - b_i)^2))
```

```python
def euclidean_distance(a, b):
    a, b = np.array(a), np.array(b)
    return np.linalg.norm(a - b)
```

### 对比

| 特性 | 余弦相似度 | 欧几里得距离 |
|------|-----------|-------------|
| 衡量维度 | 方向 | 绝对距离 |
| 是否受向量长度影响 | 否 | 是 |
| 适用场景 | 文本语义相似度 | 需要考虑向量大小时 |
| RAG 中常用 | 是（推荐） | 较少 |

## 向量数据库

向量数据库专门用于存储和检索高维向量，支持高效的近似最近邻搜索（ANN）。

### 主流向量数据库对比

| 数据库 | 类型 | 特点 | 适用场景 |
|--------|------|------|----------|
| **FAISS** | 库（非服务） | Meta 开源，性能极高 | 大规模离线检索 |
| **ChromaDB** | 嵌入式 | 零配置，API 简洁 | 快速原型、本地开发 |
| **Milvus** | 分布式服务 | 支持十亿级向量 | 生产环境、大规模部署 |
| **Qdrant** | 服务端 | Rust 实现，性能优秀 | 生产环境 |
| **Weaviate** | 服务端 | 内置向量化模块 | 全托管方案 |
| **Pinecone** | 云服务 | 全托管，零运维 | 不想自建基础设施 |

### ChromaDB 快速示例

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection(
    name="my_docs",
    metadata={"hnsw:space": "cosine"},  # 使用余弦相似度
)

# 存入文档（ChromaDB 可以自动生成 embedding）
collection.add(
    ids=["1", "2", "3"],
    documents=["Python 教程", "机器学习入门", "深度学习实战"],
)

# 检索
results = collection.query(query_texts=["如何学习 Python"], n_results=2)
print(results["documents"])
```

## 分块策略（Chunking）

将长文档切分为小段落是 RAG 的关键步骤。分块质量直接影响检索效果。

### 常见分块策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **固定长度分块** | 按字符数/Token 数切分 | 简单、可控 | 可能切断语义 |
| **语义分块** | 按段落/句子/主题切分 | 保留语义完整性 | 实现复杂 |
| **递归分块** | 逐层尝试不同分隔符 | 平衡语义和长度 | 需要调参 |
| **滑动窗口** | 固定长度 + 重叠部分 | 减少信息丢失 | 增加存储量 |

### 分块代码示例

```python
def fixed_chunk(text, chunk_size=500, overlap=50):
    """固定长度分块（带重叠）"""
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks = [*chunks, text[start:end]]
        start = end - overlap
    return chunks


def recursive_chunk(text, separators=None, chunk_size=500):
    """递归分块：逐层尝试不同分隔符"""
    if separators is None:
        separators = ["\n\n", "\n", "。", ".", " ", ""]

    if len(text) <= chunk_size:
        return [text]

    separator = separators[0]
    remaining_separators = separators[1:]
    parts = text.split(separator)

    chunks = []
    current = ""
    for part in parts:
        candidate = f"{current}{separator}{part}" if current else part
        if len(candidate) <= chunk_size:
            current = candidate
        else:
            if current:
                chunks = [*chunks, current]
            if len(part) > chunk_size and remaining_separators:
                chunks = [*chunks, *recursive_chunk(part, remaining_separators, chunk_size)]
            else:
                current = part
    if current:
        chunks = [*chunks, current]
    return chunks
```

### 分块参数建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| chunk_size | 200-1000 字符 | 太小丢失上下文，太大引入噪音 |
| overlap | chunk_size 的 10-20% | 防止关键信息在边界被切断 |
| 分隔符优先级 | 段落 > 句子 > 逗号 > 空格 | 优先保留大粒度语义 |

## 检索质量优化

### 混合搜索（Hybrid Search）

结合**关键词搜索**（BM25）和**语义搜索**（向量检索），取长补短。

```
最终得分 = α × 语义搜索得分 + (1 - α) × 关键词搜索得分
```

- 语义搜索：理解"意思"，找到语义相近但用词不同的内容
- 关键词搜索：精确匹配专有名词、代码、ID 等

### 重排序（Re-ranking）

先用向量检索召回较多候选文档（如 Top-20），再用精排模型对候选结果重新排序，取 Top-K。

```
用户提问
    ↓
粗排：向量检索 Top-20
    ↓
精排：Re-ranker 模型重新打分
    ↓
取 Top-5 送入 LLM
```

常用 Re-ranker 模型：
- `bge-reranker-large`（BAAI，开源）
- `Cohere Rerank`（API 服务）

### 其他优化技巧

| 技巧 | 说明 |
|------|------|
| 查询改写 | 用 LLM 将用户问题改写为更适合检索的形式 |
| 多路召回 | 同时用多种检索策略，合并去重 |
| 上下文压缩 | 去掉检索结果中与问题无关的部分 |
| 元数据过滤 | 利用文档的分类、时间等元数据预筛选 |
