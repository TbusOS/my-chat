## 第一步：文档加载

将原始文档读取为纯文本。支持多种格式：

```python
from pathlib import Path


def load_text_file(file_path):
    """加载纯文本文件"""
    return Path(file_path).read_text(encoding="utf-8")


def load_directory(dir_path, extensions=None):
    """加载目录下所有文档"""
    if extensions is None:
        extensions = [".txt", ".md"]
    documents = []
    for ext in extensions:
        for file_path in Path(dir_path).rglob(f"*{ext}"):
            content = file_path.read_text(encoding="utf-8")
            documents = [*documents, {
                "content": content,
                "source": str(file_path),
            }]
    return documents
```

## 第二步：文本分块

将长文档切分为适合检索的小段落：

```python
def chunk_text(text, chunk_size=500, overlap=50):
    """将文本按固定长度分块，带重叠"""
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunks = [*chunks, text[start:end]]
        start = end - overlap
    return chunks


def chunk_by_separator(text, separator="\n\n", max_size=800):
    """按段落分块，超长段落再细分"""
    paragraphs = text.split(separator)
    chunks = []
    current = ""
    for para in paragraphs:
        candidate = f"{current}{separator}{para}" if current else para
        if len(candidate) <= max_size:
            current = candidate
        else:
            if current:
                chunks = [*chunks, current]
            current = para if len(para) <= max_size else ""
            if len(para) > max_size:
                chunks = [*chunks, *chunk_text(para, max_size)]
    if current:
        chunks = [*chunks, current]
    return chunks
```

## 第三步：向量化与存储

将文本块转换为向量并存入 ChromaDB：

```python
import ollama
import chromadb


def create_knowledge_base(documents, collection_name="knowledge"):
    """创建向量知识库

    Args:
        documents: 文档列表，每项包含 content 和 source
        collection_name: 集合名称

    Returns:
        ChromaDB collection
    """
    client = chromadb.PersistentClient(path="./chroma_db")
    collection = client.get_or_create_collection(
        name=collection_name,
        metadata={"hnsw:space": "cosine"},
    )

    all_chunks = []
    all_ids = []
    all_metadatas = []
    all_embeddings = []

    for doc in documents:
        chunks = chunk_text(doc["content"])
        for i, chunk in enumerate(chunks):
            chunk_id = f"{doc['source']}_{i}"
            embedding = ollama.embed(
                model="nomic-embed-text",
                input=chunk,
            )
            all_chunks = [*all_chunks, chunk]
            all_ids = [*all_ids, chunk_id]
            all_metadatas = [*all_metadatas, {"source": doc["source"]}]
            all_embeddings = [*all_embeddings, embedding["embeddings"][0]]

    # 批量写入
    collection.add(
        ids=all_ids,
        documents=all_chunks,
        metadatas=all_metadatas,
        embeddings=all_embeddings,
    )
    print(f"已索引 {len(all_chunks)} 个文本块")
    return collection
```

## 第四步：检索与生成

用户提问时，检索相关文档并生成回答：

```python
def query_knowledge_base(collection, question, n_results=3):
    """从知识库检索相关文档"""
    q_embedding = ollama.embed(
        model="nomic-embed-text",
        input=question,
    )
    results = collection.query(
        query_embeddings=[q_embedding["embeddings"][0]],
        n_results=n_results,
    )
    return results


def generate_answer(question, context_docs):
    """基于检索结果生成回答"""
    context = "\n---\n".join(context_docs)
    prompt = (
        "你是一个知识库助手。请根据以下参考资料回答用户的问题。\n"
        "如果参考资料中没有相关信息，请如实告知。\n\n"
        f"参考资料：\n{context}\n\n"
        f"用户问题：{question}\n\n"
        "请用中文回答："
    )
    response = ollama.chat(
        model="qwen2.5",
        messages=[{"role": "user", "content": prompt}],
    )
    return response["message"]["content"]
```

## 完整使用流程

```python
# 1. 加载文档
documents = load_directory("./my_docs")
print(f"加载了 {len(documents)} 个文档")

# 2. 构建知识库
collection = create_knowledge_base(documents)

# 3. 交互式问答
while True:
    question = input("\n请输入问题（输入 q 退出）：")
    if question.strip().lower() == "q":
        break

    # 检索
    results = query_knowledge_base(collection, question)
    context_docs = results["documents"][0]

    # 显示检索到的文档来源
    print("\n检索到的相关文档：")
    for i, meta in enumerate(results["metadatas"][0]):
        distance = results["distances"][0][i]
        print(f"  [{i+1}] {meta['source']} (相似度: {1 - distance:.3f})")

    # 生成回答
    answer = generate_answer(question, context_docs)
    print(f"\n回答：{answer}")
```

## 效果优化建议

### 1. 分块策略优化

| 优化方向 | 具体做法 |
|----------|----------|
| 调整分块大小 | 技术文档用 300-500 字，叙事文档用 500-800 字 |
| 增加重叠 | 设置 10-20% 的重叠，防止关键信息被切断 |
| 语义分块 | 按段落/章节分块，保持语义完整性 |

### 2. 检索优化

| 优化方向 | 具体做法 |
|----------|----------|
| 增加 Top-K | 检索更多候选文档（如 Top-5），提高召回率 |
| 元数据过滤 | 添加分类/标签元数据，预筛选文档范围 |
| 查询改写 | 用 LLM 将口语化问题改写为更正式的检索查询 |

### 3. 生成优化

| 优化方向 | 具体做法 |
|----------|----------|
| 优化 Prompt | 明确指示模型引用来源、不编造信息 |
| 多轮对话 | 维护对话历史，支持追问 |
| 答案验证 | 检查回答是否确实基于检索到的内容 |

### 4. 使用更好的模型

```bash
# 更强的对话模型
ollama pull llama3.1
ollama pull deepseek-r1

# 中文优化的 Embedding 模型
ollama pull shaw/dmeta-embedding-zh
```

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 检索结果不相关 | 分块太大或 Embedding 模型不适合 | 减小 chunk_size，换用中文 Embedding 模型 |
| 回答出现幻觉 | Prompt 不够严格 | 在 Prompt 中强调"仅基于参考资料回答" |
| 索引速度慢 | 文档量大，逐条调用 Embedding | 使用批量 Embedding 接口 |
| 内存不足 | ChromaDB 默认全量加载 | 使用 PersistentClient 持久化存储 |
