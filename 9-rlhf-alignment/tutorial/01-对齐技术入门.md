# 对齐技术入门

## 为什么需要对齐

大语言模型（LLM）在预训练阶段学会了一件事：**预测下一个 token**。

这意味着预训练后的模型：

- 会补全文本，但不会"回答问题"
- 可能生成有害、偏见或虚假内容
- 无法拒绝不当请求
- 输出风格不可控

**对齐（Alignment）** 的目标是让模型的行为符合人类意图，具体包含三个维度：

| 维度 | 含义 | 举例 |
|------|------|------|
| 有帮助 (Helpful) | 准确完成用户请求 | 回答编程问题、总结文章 |
| 诚实 (Honest) | 不编造事实，承认不确定性 | "我不确定" 而非胡编 |
| 无害 (Harmless) | 拒绝有害请求 | 拒绝生成恶意代码 |

## 从 GPT-3 到 ChatGPT：关键区别

GPT-3（2020）和 ChatGPT（2022）的核心差异不在模型架构，而在 **对齐**。

```
GPT-3（预训练模型）
├── 输入: "写一首关于春天的诗"
└── 输出: "春天的诗有很多种，例如唐诗中..."  (补全文本，而非写诗)

ChatGPT（对齐后的模型）
├── 输入: "写一首关于春天的诗"
└── 输出: "春风拂面花满枝，细雨如丝润大地..."  (直接创作)
```

OpenAI 在 InstructGPT 论文中证明：经过对齐的 **1.3B 参数模型**，用户偏好度超过了未对齐的 **175B 参数模型**。这说明对齐的价值远超单纯的规模扩展。

## 对齐的三个阶段

InstructGPT / ChatGPT 使用的经典 RLHF 流程包含三个阶段：

### 阶段 1：SFT（Supervised Fine-Tuning）

用人工标注的高质量对话数据微调预训练模型。

```python
# SFT 数据格式示例
sft_data = [
    {
        "instruction": "解释什么是量子计算",
        "response": "量子计算是一种利用量子力学原理进行计算的技术..."
    },
    {
        "instruction": "用 Python 实现快速排序",
        "response": "def quicksort(arr):\n    if len(arr) <= 1:\n        return arr\n    ..."
    }
]
```

### 阶段 2：RM（Reward Model）

训练一个奖励模型，学习人类对回答质量的偏好排序。

```
问题: "如何学习编程？"

回答 A: "学习编程需要循序渐进：1. 选择一门语言..."  (详细、有条理)
回答 B: "去学就行了。"  (简短、无帮助)

人类标注: A > B
奖励模型学习: reward(A) > reward(B)
```

### 阶段 3：PPO（强化学习优化）

用 PPO 算法优化 SFT 模型，使其生成高奖励的回答，同时用 KL 散度约束防止模型偏离太远。

```
目标 = 最大化奖励 - KL惩罚

即：让模型回答得更好，但不要变得面目全非
```

## 主流对齐方法对比

| 方法 | 是否需要奖励模型 | 训练复杂度 | 内存需求 | 效果 | 代表论文 |
|------|:---:|:---:|:---:|:---:|------|
| RLHF (PPO) | 是 | 高 | 高（需4个模型） | 强 | InstructGPT (2022) |
| DPO | 否 | 低 | 中（需2个模型） | 接近 RLHF | Rafailov et al. (2023) |
| ORPO | 否 | 最低 | 低（仅1个模型） | 良好 | Hong et al. (2024) |
| KTO | 否 | 低 | 中 | 良好 | Ethayarajh et al. (2024) |
| IPO | 否 | 低 | 中 | 良好 | Azar et al. (2023) |

### 方法选择建议

```
资源充足 + 追求最优效果 → RLHF (PPO)
资源有限 + 有偏好数据     → DPO
数据有限 + 快速迭代       → ORPO
只有好/坏标签（无配对）   → KTO
```

## InstructGPT 论文核心思路

InstructGPT 是对齐领域的奠基性工作，核心贡献：

1. **证明了对齐的必要性**：小模型对齐后优于大模型未对齐
2. **提出了三阶段流程**：SFT -> RM -> PPO 成为标准范式
3. **人类反馈是关键**：相比无监督方法，人类反馈更有效
4. **对齐税（Alignment Tax）**：对齐后模型在某些基准测试上性能略有下降，但用户体验大幅提升

### InstructGPT 数据规模

| 阶段 | 数据量 | 标注人员 |
|------|--------|---------|
| SFT | ~13,000 条 | 40 名标注员 |
| RM | ~33,000 次比较 | 40 名标注员 |
| PPO | ~31,000 条 prompt | 无需额外标注 |

关键启发：**高质量的少量数据 > 低质量的大量数据**。
