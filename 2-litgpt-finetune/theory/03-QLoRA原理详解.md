## 1.1 量化基础

### 1.1.1 什么是量化？

**量化**：将高精度浮点数转换为低精度整数。

```
FP32:  3.14159265 (4 bytes)
INT8:  31 (1 byte)
```

### 1.1.2 量化优势

| 精度 | 内存 | 压缩比 |
|------|------|--------|
| FP32 | 4 bytes | 1x |
| FP16 | 2 bytes | 2x |
| INT8 | 1 byte | 4x |
| INT4 | 0.5 byte | 8x |

---

## 1.2 QLoRA 核心思想

### 1.2.1 基本流程

```
1. 量化模型权重到 4-bit
2. LoRA 训练时反量化
3. 计算梯度
4. 更新 LoRA 参数
5. 量化权重保持不变
```

### 1.2.2 创新点

- **4-bit NormalFloat 量化**：更精确的 4-bit 表示
- **双重量化**：进一步压缩
- **分页优化器**：减少显存

---

## 1.3 双重量化 (Double Quantization)

### 1.3.1 原理

```
第一层: FP16 → INT4
第二层: INT4 的量化参数 → INT8
```

### 1.3.2 效果

额外节省 ~0.4 bits per parameter

---

## 1.4 分页优化器 (Paged Optimizers)

### 1.4.1 问题

Adam 优化器需要保存两个 momentum 状态：

```
FP32 状态 1: 4 bytes × 参数数量
FP32 状态 2: 4 bytes × 参数数量
```

### 1.4.2 解决

使用 CPU 内存作为临时分页：

```
GPU 显存不足时 → 自动切换到 CPU
训练完成后 → 同步回 GPU
```

## 1.5 QLoRA vs LoRA vs 全参数

| 方法 | 显存 (7B) | 效果 |
|------|-----------|------|

| LoRA | 12GB | ~95% |
| QLoRA | 6GB | ~90% |
