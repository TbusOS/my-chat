---

## 1.1 背景与动机

### 1.1.1 全参数微调的问题

传统微调需要更新所有参数：

```
预训练模型: 70亿参数
微调时需要更新: 70亿参数
显存需求: 140GB+ (fp16)
```

问题：
- 显存要求高
- 训练时间长
- 部署成本大

### 1.1.2 参数效率问题

研究发现：
- 微调后模型与预训练模型的区别是**低秩**的
- 不需要显式更新所有参数

---

## 1.2 LoRA 核心思想

### 1.2.1 核心假设

**低秩假设**：模型权重的变化可以用低秩矩阵近似。

```
原始权重变化: ΔW (d × k)
LoRA 近似: ΔW ≈ BA
         其中 B (d × r), A (r × k)
         r << min(d, k)
```

### 1.2.2 直观理解

```
传统微调: W → W + ΔW (完全更新)

LoRA:    W → W + BA (只训练 A 和 B)
```

- W: 预训练权重（冻结）
- A, B: 新训练的低秩矩阵

---

## 1.3 数学推导

### 1.3.1 前向传播

```
输入: x (batch, d)

原始: h = Wx
LoRA: h = Wx + BAx
```

### 1.3.2 梯度更新

```
LoRA 参数: A, B
目标: 最小化下游任务损失

∂L/∂A = (∂L/∂h) · x^T · B^T
∂L/∂B = (∂L/∂h) · (Ax)^T
```

### 1.3.3 参数数量

| 方式 | 参数量 | 显存 |
|------|--------|------|
| 全参数 | d × k | 高 |
| LoRA | r × (d + k) | 低 |

**示例** (d=4096, k=4096, r=8):
- 全参数: 16,777,216
- LoRA: 65,536
- 减少: 256 倍

## 1.4 LoRA 架构

### 1.4.1 应用位置

LoRA 可以应用于：
- Attention 的 Q、K、V 投影
- FFN 的两个线性层

```
Transformer:
  Attention:
    Q = W_Q x + B_Q A_Q x
    K = W_K x + B_K A_K x
    V = W_V x + B_V A_V x

  FFN:
    h = W_2(W_1 x + B_1 A_1 x)
```

### 1.4.2 秩 r 的选择

| r 值 | 效果 | 适用场景 |
|------|------|----------|
| 1-4 | 欠拟合 | 简单任务 |
| 8-16 | 平衡 | 推荐默认 |
| 32-64 | 过拟合风险 | 复杂任务 |

## 1.5 优势

### 1.5.1 效率优势

| 指标 | 全参数 | LoRA | 改善 |
|------|--------|------|------|
| 可训练参数 | 100% | <1% | 100x |
| 训练显存 | 100% | ~30% | 3x |
| 推理延迟 | 0 | 可忽略 | 无 |

### 1.5.2 灵活优势

- 可动态切换任务
- 可组合多个 LoRA
- 不影响原始模型

---

## 1.6 局限

### 1.6.1 局限

1. **性能差距**：某些任务可能不如全参数微调
2. **批量推理**：同时处理多个任务时需要切换权重
3. **秩选择**：需要调参

### 1.6.2 改进方案

- **LoRA+**: 分别为 A 和 B 设置不同学习率
- **AdaLoRA**: 动态调整秩
- **QLoRA**: 结合量化进一步压缩
