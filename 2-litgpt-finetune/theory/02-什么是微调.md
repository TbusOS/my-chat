# 什么是微调

## 本章目标

深入理解微调的概念：
1. 预训练与微调的关系
2. 迁移学习
3. 不同微调方法对比

---

## 1.1 预训练与微调

### 1.1.1 预训练阶段

```
数据: 互联网上海量文本
目标: 学习通用语言能力
输出: 基础模型 (Base Model)
```

预训练让模型学习：
- 语法规则
- 语义理解
- 世界知识
- 推理能力

### 1.1.2 微调阶段

```
数据: 特定任务数据 (几千~几万条)
目标: 适配特定任务
输出: 微调后模型 (Fine-tuned Model)
```

微调让模型：
- 遵循指令
- 特定领域表现更好
- 输出格式可控

---

## 1.2 迁移学习

### 1.2.1 什么是迁移学习？

**迁移学习**：把在一个任务上学到的知识，迁移到另一个任务。

```
ImageNet 预训练 → 迁移到医学图像识别
语言模型预训练 → 迁移到对话任务
```

### 1.2.2 为什么有效？

预训练模型已经学到了：
- 语言的底层结构
- 通用知识
- 推理能力

微调只需：
- 调整输出格式
- 适应特定领域
- 学习任务模式

---

## 1.3 微调方法对比

### 1.3.1 全参数微调

```
更新: 全部参数
优点: 效果最好
缺点: 显存需求高
```

### 1.3.2 LoRA

```
更新: 少量低秩矩阵
优点: 显存需求低
缺点: 效果略逊于全参数
```

### 1.3.3 QLoRA

```
更新: 量化+LoRA
优点: 显存需求极低
缺点: 训练时间较长
```

---

## 1.4 什么时候需要微调？

### 1.4.1 需要微调

- 需要模型遵循特定格式
- 特定领域问答
- 需要特定输出风格
- 预训练模型效果不好

### 1.4.2 不需要微调

- 通用对话
- 简单问答
- 已有合适的基础模型
