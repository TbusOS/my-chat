## 3.1 训练方式

### 3.1.1 LoRA 训练

适合：消费级 GPU，数据量适中

```bash
litgpt finetune lora \
  --model microsoft/phi-2 \
  --data alpaca \
  --train.num_epochs 3
```

### 3.1.2 QLoRA 训练

适合：显存有限，使用量化模型

```bash
litgpt finetune lora \
  --model meta-llama/Llama-2-7b-chat-hf \
  --quantize qlora \
  --data alpaca \
  --train.num_epochs 3
```

### 3.1.3 全参数训练

适合：多卡 GPU，数据量大

```bash
litgpt finetune full \
  --model meta-llama/Llama-3.2-1B \
  --data alpaca \
  --train.num_epochs 1
```

## 3.3 参数配置

### 3.3.1 常用训练参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `--train.num_epochs` | 训练轮数 | 1-5 |
| `--train.batch_size` | 批次大小 | 1-16 |
| `--train.learning_rate` | 学习率 | 1e-4 ~ 1e-5 |
| `--train.warmup_steps` | 预热步数 | 100-500 |
| `--train.gradient_accumulation_steps` | 梯度累积 | 1-8 |

### 3.3.2 LoRA 参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `--lora.r` | LoRA 秩 | 8-64 |
| `--lora.alpha` | 缩放因子 | 16-32 |
| `--lora.dropout` | Dropout | 0.05-0.1 |
| `--lora.query_key` | 在 Q/K 上应用 | true |
| `--lora.value` | 在 V 上应用 | true |

### 3.3.3 完整配置示例

```bash
litgpt finetune lora \
  --model microsoft/phi-2 \
  --data alpaca \
  --out_dir out/lora-phi2 \
  \
  --train.num_epochs 3 \
  --train.batch_size 4 \
  --train.gradient_accumulation_steps 4 \
  --train.lr 3e-4 \
  --train.warmup_steps 100 \
  --train.save_interval 500 \
  --train.log_interval 10 \
  \
  --lora.r 16 \
  --lora.alpha 32 \
  --lora.dropout 0.05 \
  --lora.query_key \
  --lora.value
```

## 3.6 常见问题

### 3.6.1 训练崩溃

```
可能原因：
1. 学习率太高 → 降低 --train.lr
2. 批次太大 → 降低 --train.batch_size
3. 数据格式错误 → 检查 JSON 格式
```

解决方案：
```bash
# 降低学习率
--train.lr 1e-5

# 降低批次
--train.batch_size 1

# 梯度累积
--train.gradient_accumulation_steps 8
```

### 3.6.2 显存不足

```
解决方案：
1. 使用量化模型
2. 减少批次大小
3. 使用梯度累积
4. 使用 LoRA/QLoRA
```

### 3.6.3 效果不好

```
可能原因：
1. 数据量太少 → 增加数据
2. 训练轮数不够 → 增加 epochs
3. 数据质量差 → 清洗数据
4. 模型太小 → 换大模型
```
