# 第一章：微调入门

## 本章目标

理解微调的基本概念，包括：
1. 什么是微调
2. 为什么要微调
3. 微调 vs 预训练
4. 什么场景需要微调

---

## 1.1 什么是微调？

### 1.1.1 通俗解释

想象一下：

**预训练** = 让一个小孩读完整个图书馆的书，成为一个"通才"
**微调** = 让这个通才专门学习医学，成为一个"医生"

微调就是在一个已经训练好的模型基础上，用特定领域的数据进行进一步训练，让模型适应特定任务。

### 1.1.2 技术解释

**微调（Fine-tuning）** 是迁移学习的一种形式：

```
预训练模型（通用能力）
     ↓
微调（特定数据）
     ↓
微调后模型（特定能力）
```

微调时：
- 可以只训练部分参数（高效微调）
- 也可以训练全部参数（全参数微调）

---

## 1.2 为什么要微调？

### 1.2.1 预训练模型的局限

预训练模型虽然能力强，但有以下局限：

| 问题 | 说明 |
|------|------|
| 知识过时 | 训练数据有截止日期 |
| 领域知识不足 | 特定领域回答不专业 |
| 风格不符 | 输出风格不符合预期 |
| 格式固定 | 不能按特定格式输出 |

### 1.2.2 微调的优势

| 优势 | 说明 |
|------|------|
| 数据需求少 | 几千条数据即可 |
| 训练快速 | 几小时完成 |
| 效果明显 | 显著提升特定任务表现 |
| 成本低 | 不需要从头训练 |

---

## 1.3 预训练 vs 微调

### 1.3.1 对比

| 对比项 | 预训练 | 微调 |
|--------|--------|------|
| **数据量** | TB级别 | GB级别 |
| **训练时间** | 数周 | 数小时 |
| **计算资源** | 需要大量GPU | 消费级GPU即可 |
| **目标** | 学习通用语言能力 | 适配特定任务 |
| **参数更新** | 从零开始 | 基于预训练权重 |

### 1.3.2 训练目标

**预训练目标**：预测下一个词
```
输入: "今天天气" → 目标: "很好"
```

**微调目标**：根据指令完成任务
```
输入: "请把以下句子翻译成英文：我爱中国" → 目标: "I love China"
```

---

## 1.4 什么场景需要微调？

### 1.4.1 典型场景

| 场景 | 示例 | 数据量 |
|------|------|--------|
| **问答助手** | 公司内部知识问答 | 1000-10000条 |
| **分类任务** | 垃圾邮件分类 | 1000-50000条 |
| **对话系统** | 客服机器人 | 5000-50000条 |
| **文本生成** | 特定风格写作 | 1000-10000条 |
| **代码助手** | 公司代码规范 | 10000+条 |

### 1.4.2 判断标准

满足以下任一条件，可能需要微调：

1. 需要模型输出特定格式
2. 需要模型回答特定领域问题
3. 需要模型符合特定风格
4. 预训练模型在此任务上效果不佳

---

## 1.5 微调方法

### 1.5.1 全参数微调

```
更新所有参数
优点：效果最好
缺点：需要大量显存
```

**适用场景**：数据量大、任务复杂

### 1.5.2 高效微调

| 方法 | 说明 | 显存需求 |
|------|------|----------|
| **LoRA** | 低秩分解 | 低 |
| **QLoRA** | 量化+LoRA | 极低 |
| **Adapter** | 插入适配器 | 低 |
| **Prefix** | 前缀调优 | 低 |

**适用场景**：资源有限、任务相对简单

---

## 1.6 LoRA 简介

### 1.6.1 核心思想

LoRA (Low-Rank Adaptation) 的核心思想：

```
原始权重: W (d × k)

LoRA添加: ΔW = BA
         其中 B (d × r), A (r × k)
         r << min(d, k)
```

只训练 A 和 B，原始权重 W 保持不变。

### 1.6.2 优势

- 参数量减少 90%+
- 训练速度提升 2-3 倍
- 效果接近全参数微调
- 可动态切换任务
