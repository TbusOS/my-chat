# 分词器入门

## 什么是分词 (Tokenization)？

分词是将一段文本拆分成更小单元（Token）的过程。Token 是模型处理文本的基本单位。

```
"我爱编程" → ["我", "爱", "编", "程"] → [123, 456, 789, 101]
```

LLM 不认识文字，只认识数字。分词器就是这两个世界之间的桥梁：

```
输入文本 ──→ 分词器编码 ──→ Token ID 序列 ──→ 模型处理
                                                    ↓
输出文本 ←── 分词器解码 ←── Token ID 序列 ←── 模型输出
```

## 为什么需要分词？

### 1. 模型只理解数字

神经网络的输入是数值向量，不是字符串。我们需要一种方式将文本映射到数字。

### 2. 控制词表大小

如果每个汉字/英文单词都是一个独立 token，词表会非常大（英语单词有几十万个）。太大的词表意味着：
- 模型参数量剧增（Embedding 层和输出层与词表大小成正比）
- 训练数据中稀有词学不好

### 3. 处理未知词

好的分词方案能把从未见过的词拆成已知的子单元，而不是标记为 `[UNK]`。

## 三种分词粒度

### 字符级 (Character-level)

```python
text = "Hello"
tokens = list(text)  # ['H', 'e', 'l', 'l', 'o']
```

| 优点 | 缺点 |
|------|------|
| 词表最小（几百个字符） | 序列太长，模型难以捕捉语义 |
| 无未知词问题 | 单个字符缺乏语义 |

### 词级 (Word-level)

```python
text = "I love programming"
tokens = text.split()  # ['I', 'love', 'programming']
```

| 优点 | 缺点 |
|------|------|
| 语义完整 | 词表巨大 |
| 序列短 | 无法处理未知词 |
| | 形态变化（run/running/runs）各占一个位置 |

### 子词级 (Subword-level) -- 当前主流

```python
# "tokenization" 可能被拆为：
tokens = ["token", "ization"]
```

| 优点 | 缺点 |
|------|------|
| 词表大小适中（3万~10万） | 需要训练分词模型 |
| 高频词保持完整 | 同一个词可能有不同的拆分方式 |
| 低频词可拆成已知子词 | |

**结论：现代 LLM 几乎全部使用子词级分词。**

## 常见分词器对比

| 分词器 | 算法 | 代表模型 | 特点 | 词表大小 |
|--------|------|----------|------|----------|
| BPE | 字节对编码 | GPT-2/3/4 | 贪心合并高频字节对 | ~50k |
| WordPiece | 似然最大化 | BERT | 与 BPE 类似，选择使似然增大最多的合并 | ~30k |
| Unigram | 概率剪枝 | T5, LLaMA | 从大词表开始，逐步移除低概率 token | ~32k |
| SentencePiece | BPE/Unigram | 多种 | 语言无关，直接处理原始字节流 | 可配置 |
| tiktoken | BPE | GPT-3.5/4 | OpenAI 实现，Rust 编写，速度极快 | ~100k |

### 中文分词的特殊性

英文有天然的空格分隔，中文没有。现代分词器通常的做法：

- **UTF-8 字节级 BPE**：将所有文本视为字节流，不依赖空格分词
- 这样中文、日文、emoji 等都能自然处理
- GPT 系列就是这种方式

## 快速体验：tiktoken

tiktoken 是 OpenAI 开源的分词器，用 Rust 编写，速度极快。

### 安装

```bash
pip install tiktoken
```

### 基本用法

```python
import tiktoken

# 获取 GPT-4 使用的编码器
enc = tiktoken.encoding_for_model("gpt-4")

# 编码：文本 → Token IDs
text = "你好，世界！Hello World!"
token_ids = enc.encode(text)
print(f"文本: {text}")
print(f"Token IDs: {token_ids}")
print(f"Token 数量: {len(token_ids)}")

# 解码：Token IDs → 文本
decoded = enc.decode(token_ids)
print(f"解码还原: {decoded}")

# 查看每个 token 对应的文本
for tid in token_ids:
    print(f"  {tid} → {repr(enc.decode([tid]))}")
```

### 对比中英文 Token 效率

```python
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4")

texts = [
    "Hello, how are you today?",
    "你好，你今天怎么样？",
    "こんにちは、お元気ですか？",
]

for text in texts:
    tokens = enc.encode(text)
    ratio = len(tokens) / len(text)
    print(f"文本: {text}")
    print(f"  字符数: {len(text)}, Token 数: {len(tokens)}, 比率: {ratio:.2f}")
    print()
```

你会发现：中文和日文每个字符消耗的 token 数通常多于英文。这是因为 GPT 的词表主要基于英文语料训练，中文字符在词表中的覆盖率较低。

## 小结

| 要点 | 说明 |
|------|------|
| 分词是什么 | 将文本拆分为 Token 的过程 |
| 为什么需要 | 模型只能处理数字 |
| 主流方案 | 子词级分词（BPE 为主） |
| 中文特殊性 | UTF-8 字节级 BPE 自然支持 |
| 推荐工具 | tiktoken（GPT 系列）、tokenizers（HuggingFace） |
