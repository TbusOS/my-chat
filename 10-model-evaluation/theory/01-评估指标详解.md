# 评估指标详解

本文深入讲解 LLM 评估中最常用的四个指标：Perplexity、BLEU、ROUGE、Pass@k。

## 1. Perplexity（困惑度）

### 直觉理解

困惑度衡量的是：**模型对一段文本有多"惊讶"**。

- 困惑度低 = 模型觉得这段文本很正常、很可预测
- 困惑度高 = 模型觉得这段文本出乎意料

打个比方：如果你读到"今天天气真好"，你不会惊讶（低困惑度）。但如果读到"今天天气真紫色"，你会很困惑（高困惑度）。

### 数学公式

给定一段文本 $W = w_1, w_2, ..., w_N$，困惑度定义为：

```
PPL(W) = exp(-1/N * sum(log P(w_i | w_1, ..., w_{i-1})))
```

其中：
- $N$ 是 token 总数
- $P(w_i | w_1, ..., w_{i-1})$ 是模型预测第 $i$ 个 token 的概率

### 代码示例

```python
import torch
import math

def compute_perplexity(model, tokenizer, text):
    """计算模型在给定文本上的困惑度"""
    inputs = tokenizer(text, return_tensors="pt")
    with torch.no_grad():
        outputs = model(**inputs, labels=inputs["input_ids"])
    return math.exp(outputs.loss.item())

# 困惑度越低，模型对这段文本的建模能力越强
# 典型值：好的语言模型在 WikiText-2 上 PPL < 20
```

### 适用与局限

| 适用场景 | 局限性 |
|----------|--------|
| 比较同一模型在不同数据上的表现 | 不同 tokenizer 的模型之间不可直接比较 |
| 语言建模能力的通用度量 | 低困惑度不等于高质量生成 |
| 检测领域适配效果 | 对开放式生成任务的区分度有限 |

## 2. BLEU（双语评估替补）

### 直觉理解

BLEU 回答的问题是：**生成的文本和参考文本有多少 n-gram 是重合的？**

就像阅卷老师检查学生的翻译作业 -- 看学生用的词和短语有多少跟标准答案一样。

### 计算方式

```
BLEU = BP * exp(sum(w_n * log(precision_n)))
```

核心步骤：
1. 计算 1-gram、2-gram、3-gram、4-gram 的精确率
2. 取几何平均
3. 乘以长度惩罚因子 BP（防止生成过短的文本得高分）

### 代码示例

```python
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction

def evaluate_bleu(reference, candidate):
    """计算 BLEU 分数"""
    ref_tokens = list(reference)  # 中文按字分
    cand_tokens = list(candidate)

    smoothie = SmoothingFunction().method1
    score = sentence_bleu(
        [ref_tokens],
        cand_tokens,
        smoothing_function=smoothie
    )
    return score

# 示例
ref = "今天天气非常好，适合出门散步"
gen = "今天的天气很好，很适合散步"
print(f"BLEU: {evaluate_bleu(ref, gen):.4f}")
```

### 适用与局限

| 适用场景 | 局限性 |
|----------|--------|
| 机器翻译质量评估 | 不考虑语义等价（同义词得 0 分） |
| 文本生成的基线对比 | 对长文本的区分度下降 |
| 大规模自动评估 | 单一参考可能不公平 |

## 3. ROUGE（面向召回的摘要评估）

### 直觉理解

ROUGE 和 BLEU 相似，但关注点不同：**参考文本中有多少内容被生成的摘要覆盖了？**

BLEU 侧重精确率（生成的对不对），ROUGE 侧重召回率（该有的有没有）。

### 常见变体

| 变体 | 计算方式 | 说明 |
|------|----------|------|
| ROUGE-1 | 1-gram 召回率 | 单词级重合 |
| ROUGE-2 | 2-gram 召回率 | 短语级重合 |
| ROUGE-L | 最长公共子序列 | 考虑句子结构 |

### 代码示例

```python
from rouge_score import rouge_scorer

def evaluate_rouge(reference, generated):
    """计算 ROUGE 分数"""
    scorer = rouge_scorer.RougeScorer(
        ['rouge1', 'rouge2', 'rougeL'],
        use_stemmer=True
    )
    scores = scorer.score(reference, generated)

    return {
        metric: {
            'precision': s.precision,
            'recall': s.recall,
            'f1': s.fmeasure
        }
        for metric, s in scores.items()
    }

# 示例
ref = "The cat sat on the mat and looked out the window"
gen = "The cat was sitting on the mat"
results = evaluate_rouge(ref, gen)
for metric, values in results.items():
    print(f"{metric}: F1={values['f1']:.4f}")
```

### 适用与局限

| 适用场景 | 局限性 |
|----------|--------|
| 文本摘要评估 | 同样不考虑语义等价 |
| 问答系统评估 | 对中文效果不如英文 |
| 信息抽取评估 | 短文本区分度低 |

## 4. Pass@k（代码生成评估）

### 直觉理解

Pass@k 回答的问题是：**让模型生成 k 个代码方案，至少有一个能通过测试的概率是多少？**

这特别适合代码生成，因为同一个问题可以有多种正确实现。

### 计算公式

```
Pass@k = 1 - C(n-c, k) / C(n, k)
```

其中：
- $n$ = 总共生成的样本数
- $c$ = 通过测试的样本数
- $k$ = 从中选取的样本数

### 代码示例

```python
import numpy as np
from math import comb

def pass_at_k(n, c, k):
    """
    计算 Pass@k

    n: 总生成数
    c: 通过测试用例的数量
    k: 选取的样本数
    """
    if n - c < k:
        return 1.0
    return 1.0 - comb(n - c, k) / comb(n, k)

# 示例：生成 20 个方案，其中 5 个通过测试
n, c = 20, 5
for k in [1, 5, 10]:
    score = pass_at_k(n, c, k)
    print(f"Pass@{k}: {score:.4f}")

# 输出：
# Pass@1:  0.2500
# Pass@5:  0.7365
# Pass@10: 0.9615
```

### 适用与局限

| 适用场景 | 局限性 |
|----------|--------|
| 代码生成能力评估 | 依赖测试用例的质量和覆盖率 |
| 不同模型的代码能力对比 | 只关心"能不能跑通"，不关心代码质量 |
| 评估采样策略的效果 | 需要大量生成样本，计算成本高 |

## LLM 评估的特殊挑战

传统 NLP 指标在 LLM 时代面临新问题：

### 1. 开放式生成难以量化

```
问题：写一首关于春天的诗
参考：春风又绿江南岸，明月何时照我还
模型：桃花潭水深千尺，不及汪伦送我行  （也很好，但 BLEU=0）
```

### 2. 数据污染

模型可能在预训练时见过评估数据集，导致分数虚高。

```
解决方案：
├── 使用更新的评估集（发布时间晚于模型训练截止日期）
├── 动态生成评估题目
└── 交叉验证 + 分析异常高分
```

### 3. 提示词敏感性

同一个模型，换一种提问方式，分数可能差很多。

```python
# 这两种提问方式可能得到差异很大的结果
prompt_v1 = "What is the capital of France?"
prompt_v2 = "Answer the following question: What is the capital of France?\nAnswer:"
```

### 4. 综合能力难以用单一指标衡量

```
一个好的 LLM 需要同时具备：
├── 知识广度 (MMLU)
├── 推理能力 (GSM8K, ARC)
├── 代码能力 (HumanEval)
├── 指令跟随 (IFEval)
├── 安全性 (TruthfulQA)
└── ...没有一个指标能全部覆盖
```

## 指标选择速查表

| 你的任务 | 首选指标 | 备选指标 |
|----------|---------|---------|
| 语言建模 | Perplexity | - |
| 机器翻译 | BLEU | METEOR, COMET |
| 文本摘要 | ROUGE-L | ROUGE-2, BERTScore |
| 代码生成 | Pass@k | CodeBLEU |
| 问答系统 | F1 / EM | ROUGE-L |
| 通用能力 | MMLU Accuracy | 多基准综合 |
| 中文能力 | C-Eval / CMMLU | - |
