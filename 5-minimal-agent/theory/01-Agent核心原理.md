---

## 1.1 什么是 Agent？

### 1.1.1 Agent vs LLM

| 对比 | LLM | Agent |
|------|-----|-------|
| 能力 | 生成文本 | 执行动作 |
| 交互 | 被动响应 | 主动行动 |
| 工具 | 无 | 有 |
| 记忆 | 有限 | 长期 |

### 1.1.2 Agent 本质

```
Agent = LLM + 工具 + 记忆 + 循环
```

---

## 1.2 Agent 工作流程

### 1.2.1 ReAct 模式

```
思考 (Thought) → 行动 (Action) → 观察 (Observation)

不断循环直到完成任务
```

### 1.2.2 流程图

```
用户输入
    ↓
LLM 判断意图
    ↓
是否需要工具？
    ↓ 是 ↓
执行工具 → 返回结果
    ↓
LLM 生成回复
    ↓
返回给用户
```

## 1.3 工具调用机制

### 1.3.1 Function Calling

```json
// LLM 返回的工具调用
{
  "tool_calls": [
    {
      "function": {
        "name": "get_weather",
        "arguments": "{\"city\": \"北京\"}"
      }
    }
  ]
}
```

### 1.3.2 执行流程

```
1. 用户: "北京天气怎么样？"
2. LLM: 需要调用 get_weather(city="北京")
3. 系统: 执行 get_weather，返回 "晴，25°C"
4. LLM: "北京今天天气晴朗，气温25°C"
5. 返回给用户
```

## 1.4 思维链 (CoT)

### 1.4.1 什么是 CoT？

让 LLM 先思考再回答：

```
普通: 2 + 2 = 4

CoT:
第一步: 2 + 2
第二步: 4
结果: 4
```

### 1.4.2 Agent 中的 CoT

```
用户: 帮我买一杯咖啡

思考:
1. 用户想要买咖啡
2. 需要调用咖啡店 API
3. 需要获取用户位置
4. 需要下单并支付
→ 执行: get_location()
→ 执行: order_coffee()
→ 执行: payment()
→ 完成
```
