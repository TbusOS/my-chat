## 1.1 从字符串到数字

### 1.1.1 核心问题

计算机只能处理数字，但我们要处理的是文字（文本）。

**解决思路**：把文字转换成数字表示。

```
文本: "今天天气很好"
    ↓
数字: [1, 5, 23, 8, 102, 45, 89]
```

### 1.1.2 转换流程

```
原始文本 → 分词(Tokenization) → 数值化 → 嵌入(Embedding) → 神经网络处理
```

## 1.2 分词（Tokenization）

### 1.2.1 什么是分词？

分词是将文本拆分成更小单元的过程。

**分词方式**：
- **词级分词**：按单词分割（英文）
- **字符级分词**：按字符分割
- **子词分词**：BPE、WordPiece、SentencePiece

### 1.2.2 子词分词（主流）

现代 LLM 通常使用**子词分词**，平衡词表大小和表达能力：

```
原词: "unhappiness"

分词结果: ["un", "happiness"]
```

```
原句: "我在学习深度学习"

分词结果: ["我", "在", "学习", "深度", "学习"]
```

### 1.2.3 Token 是什么？

**Token** 是分词后的基本单元：
- 可能是一个词
- 可能是一个子词
- 可能是一个字符

```
Token 数量 ≠ 单词数量

"hello world" → 2 个单词，但可能 = 3 个 token
```

### 1.2.4 查看分词结果

```python
# 使用 TikToken（OpenAI 的分词器）
import tiktoken

enc = tiktoken.get_encoding("cl100k_base")
tokens = enc.encode("Hello, world!")
print(tokens)  # [9906, 11, 1917, 0]
print(len(tokens))  # 4
```

### 1.2.5 Token 的意义

- **输入限制**：模型有最大 Token 限制（如 4096、8192）
- **计费基础**：API 通常按 Token 收费
- **计算基础**：模型计算量与 Token 数成正比

## 1.3 词嵌入（Word Embedding）

### 1.3.1 什么是词嵌入？

把每个 Token 映射成一个固定长度的**向量**。

```
Token: "猫" → Embedding: [0.2, -0.5, 0.8, 0.1, ...] (假设 768 维)

Token: "狗" → Embedding: [0.3, -0.4, 0.7, 0.2, ...]
```

### 1.3.2 为什么需要嵌入？

**原因**：直接用 One-Hot 编码有两个问题：
1. 维度太高（词表有多大，维度就多大）
2. 无法表达语义相似性

```
One-Hot (假设词表 10000):
"猫" → [1, 0, 0, 0, ..., 0]
"狗" → [0, 1, 0, 0, ..., 0]

问题：猫和狗的向量正交，无法表示它们相似
```

**解决**：用低维稠密向量

```
Embedding (假设 256 维):
"猫" → [0.2, -0.5, 0.8, ...]
"狗" → [0.3, -0.4, 0.7, ...]

优点：可以用距离衡量语义相似度
```

### 1.3.3 语义空间

嵌入后的向量构成了一个**语义空间**：

```
语义空间示意图（简化到2维）:

        动物
          ↑
    猫 ←  → 狗
          |
    人 ←  → 车
          |
         工具
```

- **距离近**：语义相似（猫 ↔ 狗）
- **方向同**：关系类似（国王 ↔ 男人 ≈ 女王 ↔ 女人）

### 1.3.4 嵌入矩阵

```python
# 假设
vocab_size = 10000  # 词表大小
embedding_dim = 256  # 嵌入维度

# 嵌入矩阵：vocab_size × embedding_dim
embedding_matrix =随机初始化

# 查表
token_id = 1234  # "猫" 的 ID
embedding = embedding_matrix[token_id]  # 获取嵌入向量
```

## 1.4 语言模型

### 1.4.1 什么是语言模型？

**语言模型**是一个学习**概率分布**的模型：

```
P(文本) = P(w1) × P(w2|w1) × P(w3|w1,w2) × ...
```

**本质**：给定前文，预测下一个词的概率。

### 1.4.2 下一个词预测

```python
# 简化示例
输入: "今天天气"
输出: 预测下一个词的概率

{
    "很好": 0.3,
    "不错": 0.2,
    "晴朗": 0.15,
    "很差": 0.1,
    ...
}
```

### 1.4.3 自回归生成

LLM 采用**自回归**方式生成：

```
Step 1: 输入 "今天天气" → 预测 "很好" → 输出 "很好"
Step 2: 输入 "今天天气很好" → 预测 "，" → 输出 "，"
Step 3: 输入 "今天天气很好，" → 预测 "适合" → 输出 "适合"
... 重复直到结束
```

### 1.4.4 生成过程

```python
# 伪代码：自回归生成
def generate(prompt, max_tokens=100):
    input_ids = tokenize(prompt)

    for _ in range(max_tokens):
        # 1. 获取下一个 token 的概率分布
        logits = model(input_ids)

        # 2. 采样（可以用不同策略）
        next_token = sample(logits)

        # 3. 添加到输入
        input_ids.append(next_token)

        # 4. 如果是结束符则停止
        if next_token == EOS:
            break

    return detokenize(input_ids)
```

## 1.5 Transformer 基础

### 1.5.1 整体架构

Transformer 是现代 LLM 的核心架构：

```
输入嵌入 → 位置编码 → Encoder × N → 输出
              ↓
        Self-Attention（自注意力）
              ↓
        前馈网络
```

### 1.5.2 核心组件

1. **自注意力（Self-Attention）**
   - 让每个词"关注"句子中的其他词
   - 理解上下文关系

2. **前馈网络（FFN）**
   - 进一步处理和转换特征

3. **残差连接（Residual）**
   - 缓解深层网络训练困难

4. **层归一化（LayerNorm）**
   - 稳定训练

### 1.5.3 自注意力机制

**核心问题**：如何让每个词理解上下文？

```
"银行" 在不同句子中意思不同：
- "我去银行存钱"（金融机构）
- "河边的银行"（河岸）

需要根据上下文确定含义
```

**解决方案**：自注意力

```python
# 自注意力核心公式
Attention(Q, K, V) = softmax(QK^T / √d_k) V

# Q (Query): 我当前要找什么
# K (Key): 别人有什么
# V (Value): 别人具体的内容
```

## 1.6 模型训练

### 1.6.1 训练目标

LLM 的训练目标：**预测下一个词**。

```
输入: "今天天气" → 目标: "很好"
```

### 1.6.2 训练过程

```
1. 准备大量文本数据（互联网文本）
2. 把文本分成 Token
3. 训练神经网络预测下一个 Token
4. 通过反向传播调整参数
5. 迭代直到收敛
```

### 1.6.3 预训练 vs 微调

| 阶段 | 数据 | 目的 |
|------|------|------|
| **预训练** | 海量互联网文本 | 学习通用语言能力 |
| **微调** | 特定领域数据 | 适配特定任务 |
