## 1.1 什么是 LLM？

### 1.1.1 通俗解释

**LLM (Large Language Model)** 即大语言模型，你可以把它想象成一个：

- **超级图书管理员**：它读过了互联网上几乎所有的文本，记住海量知识
- **语言天才**：精通各种语言，能理解和生成自然语言
- **问答机器**：能回答问题、写作文、写代码、翻译

当你对它说"帮我写一首诗"，它会分析这句话的意思，然后根据它"看过"的无数首诗，生成一首新的诗。

### 1.1.2 技术解释

从技术角度，LLM 是一个基于**深度学习**的**神经网络**模型：

```
输入文本 → 分词(Tokenization) → 嵌入(Embedding) → 神经网络 → 预测下一个词 → 输出文本
```

简单来说，LLM 的核心能力是：**给定前面的文字，预测下一个最可能的文字**。

### 1.1.3 LLM 能做什么？

| 能力 | 示例 |
|------|------|
| 对话 | "你好，我们来聊聊" |
| 写作 | "帮我写一封邮件" |
| 编程 | "写一个快速排序函数" |
| 翻译 | "把这段话翻译成英文" |
| 总结 | "帮我总结这篇文章" |
| 推理 | "如果A>B，B>C，那么A>C" |

## 1.2 为什么要本地运行？

### 1.2.1 云端 API vs 本地运行

| 对比项 | 云端 API (如 ChatGPT) | 本地运行 (Ollama) |
|--------|----------------------|-------------------|
| **隐私** | 数据发送到第三方服务器 | 数据完全留在本地 |
| **网络** | 需要联网 | 无需联网也能用 |
| **费用** | 按 token 收费 | 一次性投入硬件 |
| **定制** | 无法微调 | 可以自由微调 |
| **速度** | 依赖网络延迟 | 本地更快 |
| **可用性** | 可能服务不稳定 | 完全自己掌控 |

### 1.2.2 本地运行的优势

1. **隐私安全**：公司机密、个人隐私数据不离开你的电脑
2. **离线可用**：飞机上、地下室、没有网络的场景都能用
3. **无限使用**：不按 token 计费，想用多少用多少
4. **可定制**：可以微调成专属模型，比如只回答你公司的产品问题

### 1.2.3 谁适合本地运行？

- 开发者：构建自己的 AI 应用
- 企业：数据隐私要求高
- 爱好者：学习 LLM 技术
- 研究者：做实验、跑测试

## 1.3 Ollama 是什么？

### 1.3.1 简介

**Ollama** 是一个让你在本地轻松运行大模型的工具：

- **一键安装**：安装简单，跨平台支持
- **模型管理**：命令行管理模型，下载/删除很简单
- **多模型支持**：Llama、Qwen、Mistral 等主流开源模型
- **API 友好**：提供 Python/JS API，方便开发

### 1.3.2 Ollama 支持的模型

| 模型 | 特点 | 推荐场景 |
|------|------|----------|
| **Qwen2.5** | 中文能力最强 | 中文项目首选 |
| **Llama 3** | Meta 出品，英文强 | 英文对话 |
| **Mistral** | 欧洲团队，推理快 | 通用场景 |
| **Phi-3** | 微软，轻量快速 | 低配电脑 |
| **DeepSeek-Coder** | 编程能力强 | 代码助手 |
| **Gemma** | Google 出品 | 轻量场景 |

## 1.4 硬件要求

### 1.4.1 配置要求

| 配置 | 最低要求 | 推荐配置 |
|------|----------|----------|
| **内存** | 8GB | 16GB+ |
| **存储** | 20GB | 50GB+ SSD |
| **显卡** | 无（CPU运行） | NVIDIA 8GB+ 或 Apple Silicon |
| **速度** | 慢 | 流畅 |

### 1.4.2 显存与模型对应

| 显存大小 | 能跑的模型 | 速度 |
|----------|-----------|------|
| 无 GPU | 3B 以下模型 | 很慢 |
| 8GB | 7B 模型(量化版) | 慢 |
| 16GB | 7B/8B 模型 | 中等 |
| 24GB | 13B 模型 | 较快 |
| 40GB+ | 70B 模型 | 快 |

**初学者推荐**：先试 7B 模型（如 qwen2.5:7b），资源要求低，效果也不错。

### 1.4.3 检查自己的电脑

**macOS**：
```bash
# 查看内存
top -l 1 | head -5

# 查看 CPU
sysctl -n machdep.cpu.brand_string

# 查看 Apple Silicon
system_profiler SPHardwareDataType | grep Chip
```

**Linux**：
```bash
free -h           # 查看内存
nvidia-smi        # 查看 NVIDIA 显卡（如果有）
```

**Windows**：
```
右键"此电脑" → 属性 查看内存
设备管理器 → 显示适配器 查看显卡
```

## 1.5 安装 Ollama

### 1.5.1 macOS 安装

**方法一：官网下载**

1. 打开 https://ollama.com
2. 点击 Download
3. 下载 macOS 版本
4. 双击安装，按提示操作

**方法二：命令行安装**

```bash
# 打开终端，运行：
curl -fsSL https://ollama.com/install.sh | sh
```

安装过程中可能需要输入密码（管理员权限）。

### 1.5.2 Linux 安装

```bash
# 安装
curl -fsSL https://ollama.com/install.sh | sh

# 验证安装
ollama --version

# 启动服务（通常自动启动）
ollama serve
```

**常见问题**：

如果安装失败，尝试：
```bash
# 1. 安装依赖
sudo apt update
sudo apt install curl build-essential

# 2. 手动下载安装脚本
wget https://ollama.com/install.sh
chmod +x install.sh
./install.sh
```

### 1.5.3 Windows 安装

**方法一：官网下载（推荐）**

1. 打开 https://ollama.com/download/windows
2. 下载安装包
3. 双击安装

**方法二：WSL2（性能更好）**

```bash
# 1. 启用 WSL2（以管理员身份运行 PowerShell）
wsl --install

# 2. 进入 WSL
wsl

# 3. 安装 Ollama（同 Linux）
curl -fsSL https://ollama.com/install.sh | sh
```

### 1.5.4 验证安装

```bash
# 查看版本
ollama --version

# 启动服务（一般自动启动）
ollama serve

# 测试运行
ollama run qwen2.5 "你好"
```

## 1.6 运行第一个模型

### 1.6.1 下载模型

```bash
# 查看已下载的模型
ollama list

# 下载 qwen2.5（推荐，中文好）
ollama pull qwen2.5

# 或者指定版本
ollama pull qwen2.5:7b
ollama pull qwen2.5:3b

# 其他模型
ollama pull llama3
ollama pull mistral
ollama pull phi3
```

**下载时间**：
- 7B 模型约 4GB，首次下载 10-30 分钟（视网速）
- 3B 模型约 2GB，下载更快

### 1.6.2 交互式运行

```bash
# 启动对话
ollama run qwen2.5
```

然后你就可以直接输入文字对话了：
```
>>> 你好
你好！有什么我可以帮你的吗？

>>> 给我讲个笑话
当然可以！...

>>> /bye   # 退出
```

### 1.6.3 非交互式运行

```bash
# 直接提问（不进入交互模式）
ollama run qwen2.5 "什么是 Python？"

# 带参数运行
ollama run qwen2.5 --temp 0.5 --num 100 "解释一下机器学习"
```
