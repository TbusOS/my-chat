## 2.1 主流模型介绍

### 2.1.1 模型参数规模

在了解具体模型之前，先明白一个概念：**模型参数规模**。

模型参数规模（Parameters）是指模型神经网络的权重数量，通常用"B"表示（10亿）：

| 参数规模 | 常见表示 | 典型大小 |
|----------|----------|----------|
| 3B | 30亿 | 约 2-4 GB |
| 7B | 70亿 | 约 4-8 GB |
| 8B | 80亿 | 约 5-10 GB |
| 13B | 130亿 | 约 8-16 GB |
| 70B | 700亿 | 约 40-80 GB |

**规律**：参数越多，能力越强，但需要的资源也越多。

### 2.1.2 推荐模型对比

| 模型 | 参数 | 中文 | 英文 | 代码 | 速度 | 推荐场景 |
|------|------|------|------|------|------|----------|
| **Qwen2.5** | 3B/7B/14B | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ | 快 | 中文项目首选 |
| **Llama 3** | 8B | ⭐⭐ | ⭐⭐⭐ | ⭐⭐ | 中 | 英文对话 |
| **Mistral** | 7B | ⭐⭐ | ⭐⭐⭐ | ⭐⭐ | 快 | 通用场景 |
| **Phi-3** | 3.8B | ⭐ | ⭐⭐ | ⭐ | 最快 | 低配电脑 |
| **DeepSeek-Coder** | 6.7B | ⭐⭐ | ⭐⭐ | ⭐⭐⭐ | 中 | 编程助手 |
| **Gemma** | 2B/7B | ⭐ | ⭐⭐ | ⭐⭐ | 快 | 轻量场景 |

### 2.1.3 各模型详细介绍

#### Qwen2.5（推荐）

- **开发团队**：阿里巴巴
- **特点**：中文能力最强，对中文语境理解好
- **优势**：免费可商用，中文对话自然
- **推荐版本**：qwen2.5:7b（平衡性能和资源）

```bash

```

#### Llama 3

- **开发团队**：Meta（Facebook）
- **特点**：英文能力强，开源先驱
- **优势**：生态丰富，文档完善

```bash

```

#### Mistral

- **开发团队**：Mistral AI（法国）
- **特点**：欧洲开源标杆，推理效率高
- **优势**：Apache 许可证，完全免费商用

```bash

```

#### Phi-3

- **开发团队**：微软
- **特点**：轻量快速，适合低配置
- **优势**：笔记本也能跑

ollama pull phi3
```

#### DeepSeek-Coder

- **开发团队**：深度求索
- **特点**：编程能力强

```

## 2.2 选择合适的模型

### 2.2.1 按场景选择

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 中文对话 | qwen2.5 | 中文理解最好 |
| 英文对话 | llama3 | 英文能力最强 |
| 代码编程 | deepseek-coder | 编程专项优化 |
| 低配置电脑 | phi3 | 轻量快速 |
| 通用场景 | mistral | 平衡稳定 |

### 2.2.2 按硬件选择

| 显存 | 推荐模型 |
|------|----------|
| 无 GPU | phi3:3.8b, gemma:2b |
| 8GB | qwen2.5:3b, phi3, llama3:8b (量化) |
| 16GB | qwen2.5:7b, llama3:8b |
| 24GB | qwen2.5:14b, llama3:70b (量化) |
| 40GB+ | llama3:70b |

### 2.2.3 量化模型

如果显存不够，可以使用**量化模型**。量化会压缩模型大小，但会略微降低质量。

```bash
# 普通版本
ollama pull qwen2.5:7b

# 量化版本（更小更快）
ollama pull qwen2.5:7b-q4_K_M  # 4bit 量化
ollama pull qwen2.5:7b-q5_K_M  # 5bit 量化
```

量化版本后缀说明：
- `_q4_K_M`：4bit 量化，体积小，速度快
- `_q5_K_M`：5bit 量化，效果好一点
- `_q8_0`：8bit 量化，效果接近原版

## 2.3 模型参数详解

### 2.3.1 运行参数

使用 `ollama run` 时可以加参数：

```bash
ollama run qwen2.5 [参数] "你的问题"
```

常用参数：

| 参数 | 说明 | 常用值 |
|------|------|--------|
| `-t, --temp` | 创造性控制 | 0.1-2.0 |
| `-n, --num` | 生成token数 | 1-4096 |
| `--top-p` | 核采样 | 0.1-1.0 |
| `--top-k` | 候选词数量 | 1-100 |
| `--repeat-penalty` | 重复惩罚 | 1.0-2.0 |

### 2.3.2 参数详细解释

#### temperature（创造性）

- **作用**：控制输出的随机性
- **值域**：0.0 - 2.0
- **推荐**：
  - 0.1-0.3：精确任务（数学、代码）
  - 0.7-0.8：一般对话
  - 1.0-1.5：创意写作

```bash
# 精确回答（低创造性）
ollama run qwen2.5 -t 0.2 "1+1等于多少？"

# 创意回答（高创造性）
ollama run qwen2.5 -t 1.5 "写一个科幻故事开头"
```

#### num_predict（生成长度）

- **作用**：限制回复的最大长度
- **值域**：1 - 4096
- **注意**：越长生成越慢，消耗资源越多

```bash
# 简短回答
ollama run qwen2.5 -n 50 "什么是Python？"

# 长篇大论
ollama run qwen2.5 -n 500 "详细解释Python的特点"
```

#### top_p（核采样）

- **作用**：控制输出词汇的多样性
- **值域**：0.0 - 1.0
- **推荐**：0.9 左右
- **原理**：只从累积概率前 p% 的词中选择

```bash
ollama run qwen2.5 --top-p 0.9 "今天天气怎么样？"
```

#### repeat_penalty（重复惩罚）

- **作用**：减少重复输出
- **值域**：1.0 - 2.0
- **推荐**：1.1 左右
- **原理**：1.0 = 无惩罚，>1.0 = 惩罚重复

```bash
ollama run qwen2.5 --repeat-penalty 1.1 "给我讲一个故事"
```

## 2.4 模型管理

### 2.4.1 查看模型

```bash
# 列出所有模型
ollama list

# 查看模型详情
ollama show qwen2.5
```

### 2.4.2 删除模型

```bash
# 删除模型
ollama rm qwen2.5

# 删除所有未使用的模型
ollama prune
```

### 2.4.3 复制/重命名模型

```bash
# 复制模型
ollama cp qwen2.5 my-qwen

# 查看模型信息
ollama show my-qwen
```

### 2.4.4 导入模型

```bash
# 从 GGUF 文件导入
ollama import model.gguf
```

## 2.5 实战技巧

### 2.5.1 常用命令速查

```bash
# 查看已下载模型
ollama list

# 下载模型
ollama pull 模型名

# 删除模型
ollama rm 模型名

# 运行模型
ollama run 模型名

# 查看模型信息
ollama show 模型名

# 复制模型
ollama cp 源模型 目标模型
```

### 2.5.2 常见问题解决

**Q: 模型下载太慢**

```bash
# 方法1：使用镜像（需要自己配置）
# 方法2：尝试更小的模型
ollama pull phi3

# 方法3：夜间下载（网络较空）
```

**Q: 回答太长/太短**

```bash
# 限制长度
ollama run qwen2.5 -n 100 "你的问题"

# 调整创造性
ollama run qwen2.5 -t 0.2 "精确回答问题"
```

**Q: 回答重复**

```bash
# 增加重复惩罚
ollama run qwen2.5 --repeat-penalty 1.5 "你的问题"
```
