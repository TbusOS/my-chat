# 常见训练问题排查

## 本章目标

解决 nanoGPT 训练中最常见的问题：
1. Loss 不降
2. 梯度爆炸/消失
3. 显存不足（OOM）
4. 生成质量差
5. 训练速度慢
6. 常见报错及修复

---

## 1. Loss 不降

| 可能原因 | 检查方法 | 解决方案 |
|----------|----------|----------|
| 学习率太小 | loss 几乎不变 | `--learning_rate=1e-3` |
| 学习率太大 | loss 震荡或 NaN | `--learning_rate=1e-4` |
| 数据损坏 | 检查 train.bin 大小 | 重新运行 prepare.py |
| 数据集太小 | token 数 < 10K | 增加数据量 |

```python
import numpy as np
# 快速诊断数据文件
data = np.memmap('data/shakespeare_char/train.bin', dtype=np.uint16, mode='r')
print(f"tokens: {len(data):,} | max ID: {data.max()} | min ID: {data.min()}")
# 如果 max ID >= vocab_size，数据有问题
```

### 学习率扫描

```bash
# 用不同学习率快速跑 500 步，找最优值
python train.py --learning_rate=1e-5 --max_iters=500 --out_dir=out-lr1
python train.py --learning_rate=6e-4 --max_iters=500 --out_dir=out-lr2
python train.py --learning_rate=1e-3 --max_iters=500 --out_dir=out-lr3
```

## 2. 梯度爆炸/消失

### 梯度爆炸（loss 变 NaN）

```bash
python train.py --learning_rate=3e-4 --grad_clip=0.5  # 降学习率 + 强裁剪
```

### 梯度消失（loss 下降极慢）

```bash
python train.py --learning_rate=1e-3 --n_layer=4 --dropout=0.0
```

### 监控梯度范数

```python
for p in model.parameters():
    if p.grad is not None:
        norm = p.grad.data.norm(2).item()
        if norm > 10.0:
            print(f"警告: 梯度过大 = {norm:.4f}")
        if norm < 1e-7:
            print(f"警告: 梯度过小 = {norm:.10f}")
```

## 3. 显存不足（OOM）

```
RuntimeError: CUDA out of memory.
```

按优先级排序的解决方案：

```bash
# 方案 1：减小 batch_size
python train.py --batch_size=16   # 从 64 减到 16

# 方案 2：梯度累积（小 batch 模拟大 batch）
python train.py --batch_size=8 --gradient_accumulation_steps=8  # 等效 bs=64

# 方案 3：减小上下文窗口
python train.py --block_size=128  # 从 256 减到 128

# 方案 4：减小模型
python train.py --n_layer=4 --n_head=4 --n_embd=256

# 方案 5：混合精度（CUDA）
python train.py --dtype=bfloat16  # 节省约 50% 显存
```

### 显存估算

| 配置 | 参数量 | 预估显存（float16） |
|------|--------|---------------------|
| 4L/4H/256E, bs=32 | ~3M | ~1 GB |
| 6L/6H/384E, bs=64 | ~10M | ~3 GB |
| 12L/12H/768E, bs=12 | ~85M | ~8 GB |

## 4. 生成质量差

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 全是重复内容 | temperature 太低 | `--temperature=0.8` |
| 输出乱码 | 训练不充分 | 增加 max_iters |
| 语法混乱 | temperature 太高 | `--temperature=0.7` |
| 内容单一 | top_k 太小 | `--top_k=200` |

```bash
# 推荐采样参数
python sample.py --temperature=0.8 --top_k=200 --out_dir=out-shakespeare
```

| temperature | top_k | 效果 |
|-------------|-------|------|
| 0.5 | 50 | 保守、重复多 |
| 0.8 | 200 | 平衡（推荐） |
| 1.0 | 500 | 多样、偶有错误 |
| 1.2 | 全部 | 混乱（不推荐） |

## 5. 训练速度慢

| 优化方法 | 加速倍数 | 适用平台 |
|----------|----------|----------|
| `torch.compile` | ~2x | Linux + CUDA |
| Flash Attention | ~2x | PyTorch 2.0+ |
| `bfloat16` 混合精度 | ~1.5x | CUDA Ampere+ |
| 增大 batch_size | ~1.2x | 显存允许时 |
| 减少评估频率 | ~1.1x | `--eval_interval=1000` |

```bash
# Linux CUDA 最大加速
python train.py --compile=True --dtype=bfloat16 --device=cuda

# 多 GPU 并行（DDP）
torchrun --standalone --nproc_per_node=4 train.py --batch_size=12
```

> Mac MPS 不支持 `torch.compile`，必须设 `--compile=False`。

## 6. 常见报错速查

| 报错 | 修复 |
|------|------|
| `FileNotFoundError: train.bin` | `python data/xxx/prepare.py` |
| `CUDA out of memory` | 见第 3 节 |
| `RuntimeError: MPS backend` | `--compile=False` |
| `ModuleNotFoundError: tiktoken` | `pip install tiktoken` |
| `ValueError: vocab_size mismatch` | 确认 dataset 和 meta.pkl 一致 |
| `loss is nan` | 降低学习率 + `--grad_clip=0.5` |

### 恢复中断的训练

```bash
python train.py --out_dir=out-shakespeare --init_from=resume --device=mps --compile=False
```

### 完整排查流程

```
训练有问题？
├── loss 是 NaN？ → 降低学习率 / 加梯度裁剪
├── loss 不降？   → 检查数据 / 调整学习率 / 增大模型
├── OOM？         → 减 batch_size / 梯度累积 / 减 block_size
├── 生成质量差？  → 训练更多步 / 调 temperature + top_k
└── 速度慢？      → torch.compile / bfloat16 / 增大 batch
```
