# 你真的会用 AI 吗？

一个扎心的事实：绝大多数人把 AI 当成了"高级搜索引擎"。

<div class="callout">

**真实数据：90% 的用户只会用 AI 做基础对话**
— 问个问题，得到答案，然后关掉。这和用百度有什么区别？

</div>

### 常见的"初级用法"

- "帮我翻译一下这段话" — 把 AI 当翻译机
- "Python 怎么读文件？" — 把 AI 当 Stack Overflow
- "写一封邮件" — 把 AI 当模板生成器

> 这些用法没有错，但你只用了 AI **不到 10%** 的能力。
> 就像你买了一台电脑，却只用它看天气预报。

# "高位截瘫的天才"

在深入之前，先建立一个关键认知 — 纯大模型**有脑无手**。

<div class="columns">
<div class="col">

### 纯大模型（ChatGPT 网页版）

**= 高位截瘫的天才**

- 脑子极其好用：推理、写作、分析、创意
- 但**没有手脚** — 不能操作你的电脑
- 不能读文件、不能跑代码、不能调接口
- 你问它，它只能"纸上谈兵"

</div>
<div class="col">

### Agent（OpenClaw / Cursor）

**= 给天才装上了机甲**

- `Agent = LLM + Tools + Memory + Planning`
- 不仅能想，还能**动手干活**
- 读写文件、执行代码、浏览网页、调用 API
- 遇到错误会**自动重试和修正**

</div>
</div>

<div class="callout">

**预告：** 第 5 章会详细讲 OpenClaw 怎么给 AI "穿机甲" — 它本身没有脑子，但提供了手脚（操作电脑的权限）。把大模型装进去，AI 就能自动干活了。

</div>

# AI 不是工具，是"人"

这是今天最重要的一句话，也是整个培训的核心理念：

<div class="callout-blue">

**别把 AI 当工具，把 AI 当人。**

工具是被动的 — 你按按钮，它执行动作。
人是主动的 — 你给背景，它理解意图、规划方案、自主完成。

</div>

### AI 能做什么"工具"做不到的事？

| 维度 | 传统工具 | AI（当人用） |
|------|---------|-------------|
| 理解能力 | 只认关键词 | 理解上下文、语义、意图 |
| 执行方式 | 固定流程 | 根据情况灵活调整 |
| 输出质量 | 千篇一律 | 因人而异、因场景而异 |
| 协作方式 | 单次输入/输出 | 多轮对话、持续迭代 |

# "聊天式" vs "当人用"

同一个需求，两种截然不同的使用方式：

<div class="columns">
<div class="col">

### 聊天式（初级）

```text
用户：帮我写个排序函数

AI：好的，这是一个冒泡排序...
```

**结果：** 得到一个教科书级别的通用代码，和你的业务毫无关系。

</div>
<div class="col">

### 当人用（高级）

```text
用户：你是资深 Go 工程师，负责
订单处理模块。需要对百万级请求日志按
时间排序，要求：
- 内存占用最小（容器限制 2GB）
- 支持增量排序（实时日志流入）
- 排序中断后可恢复
```

**结果：** 得到针对高并发场景优化的生产级代码，附带内存分析和边界处理。

</div>

# 什么是 AI？

从"规则"到"学习"，AI 的演进之路：

| 年代 | 阶段 | 核心思路 | 软件开发类比 |
|------|------|---------|---------|
| 1950s | **规则系统** | 人写 if-else 规则 | 硬编码配置：写死在代码里的业务规则 |
| 1990s | **机器学习** | 从数据中找规律 | 推荐系统：根据用户行为数据预测偏好 |
| 2012 | **深度学习** | 多层神经网络 | 图像识别：自动识别验证码和图片内容 |
| 2017 | **Transformer** | 注意力机制 | 理解整句话的含义，而不是逐字匹配 |
| 2022+ | **大语言模型** | 通用智能涌现 | 一个"什么都懂"的全能助手 |

<div class="callout-blue">

**关键转折点：** 2017 年 Google 发表论文 *"Attention Is All You Need"*，
Transformer 架构诞生，开启了今天大模型时代的序幕。

</div>

# 什么是 AGI？

**AGI = Artificial General Intelligence（通用人工智能）**

能像人类一样完成**任何**智力任务的 AI。

### 今天的 AI vs AGI

<div class="columns">
<div class="col">

**今天的 AI（窄 AI）**

- 写代码很强，但不会修水管
- 翻译很好，但不理解文化
- 下棋能赢世界冠军，但不会过马路
- 每个能力需要专门训练

</div>
<div class="col">

**AGI（通用 AI）**

- 学一个领域能迁移到另一个
- 理解抽象概念和常识
- 具备自主学习和推理能力
- 无需针对每个任务专门训练

</div>
</div>

> **现状：** 我们离 AGI 还有距离，但大语言模型展现出的"涌现能力"
> 让很多研究者认为，我们可能比想象中更接近。OpenAI 的目标就是实现 AGI。

# 什么是 ASI？

**ASI = Artificial Super Intelligence（超级人工智能）**

在**所有**智力维度上全面超越人类最优秀个体的 AI。

### 从计算器到超级智能

| 阶段 | 类比 | 与人类的关系 |
|------|------|-------------|
| 传统软件 | 计算器 | 算得快，但你告诉它算什么 |
| 今天的 AI | 聪明的实习生 | 能力强，但需要指导和监督 |
| AGI | 全能同事 | 任何智力工作都能独立完成 |
| **ASI** | **？？？** | **超越人类理解的智能水平** |

<div class="callout">

**重要提醒：** ASI 目前是纯理论概念，尚无公认的实现路径。
但它是 AI 安全研究的核心关注点 — 如果真的出现，
我们如何确保它与人类的价值观对齐？这是 Anthropic（Claude 的公司）成立的初衷。

# AI 的"大脑"：Transformer

不讲数学公式，用一个比喻来理解 Transformer 的核心 — **注意力机制**。

### 比喻：快递分拣中心

想象你在一个快递分拣中心工作：

1. **收到一批包裹（输入文本）** — 每个包裹上有地址信息（每个词有含义）
2. **查看所有包裹（自注意力）** — 分拣员不是只看一个包裹，而是**同时看所有包裹**，理解它们之间的关系（"这批都是发往上海的"、"这两个是同一个客户的"）
3. **标注优先级（注意力权重）** — 对每个包裹，判断其他包裹与它的关联程度。急件关注时效，大件关注路线
4. **分拣输出（生成结果）** — 基于对全局的理解，把每个包裹送到正确的位置

<div class="callout-blue">

**核心洞察：** 传统模型像流水线，一个一个处理。Transformer 像一个经验丰富的分拣员，一眼看全局，效率和准确率都大幅提升。这就是为什么 AI 能"理解"长文本。

</div>

# 预训练：读遍互联网

大语言模型在"上岗"之前，经历了一场超级特训。

### 类比：新员工入职

想象公司来了一个新员工，但他入职前做了一件疯狂的事：

**把公司成立以来的所有文档、邮件、会议记录、代码仓库全部读了一遍。**

这就是预训练 — AI 在和你对话之前，已经"读"过了：

| 数据来源 | 规模 | 相当于 |
|---------|------|--------|
| 互联网网页 | 数万亿 token | 数百万本书的文字量 |
| 开源代码 | GitHub 等平台 | 几乎所有公开的代码仓库 |
| 学术论文 | 数百万篇 | 大部分公开学术知识 |
| 书籍 | 数十万本 | 一座大型图书馆 |

> **GPT-4 的训练数据约 13 万亿 token。**
> 一个人一辈子能读的书大约 5000 本，GPT-4 相当于读了数十万倍。

# 为什么 AI 能"理解"你说的话？

从你打字到 AI 回复，背后发生了什么？

### 处理流水线（类比 HTTP 请求处理）

| 步骤 | AI 处理 | HTTP 类比 |
|------|---------|---------|
| **1. 接收输入** | 收到你的文本 | 服务器接收 HTTP 请求 |
| **2. Tokenization（分词）** | "你好世界" → ["你好", "世界"] | 解析请求头、路径、参数 |
| **3. Embedding（向量化）** | 每个词变成一串数字（768维向量） | 将请求参数序列化为内部数据结构 |
| **4. Attention（注意力计算）** | 理解词与词之间的关系 | 路由匹配 + 中间件逐层校验 |
| **5. Generation（生成）** | 逐字预测下一个最可能的词 | 逐步执行业务逻辑、组装响应体 |

<div class="callout-blue">

**关键理解：** AI 不是"搜索答案"，而是根据理解**逐字生成**回复。
每一个字都是基于前文的概率预测。这就是为什么同一个问题，每次回答可能略有不同。

</div>

# 多模态实际能干嘛？

别只停留在概念，看看实际应用场景：

### 与日常开发直接相关的场景

| 场景 | 输入 | AI 输出 | 价值 |
|------|------|---------|------|
| **故障诊断** | 截图报错页面或终端输出 | 分析错误原因 + 解决步骤 | 减少排障时间 |
| **文档识别** | 拍一张纸质文档照片 | 结构化数据（JSON） | 快速数字化录入 |
| **文档理解** | 上传技术手册 PDF | 回答关于手册的问题 | 新人快速上手 |

### 更广泛的应用

- **语音指令 → 代码生成：** 口述需求，AI 直接写代码
- **生产线视频 → 质检报告：** 摄像头拍视频，AI 检测不良品
- **手绘草图 → 原型设计：** 在纸上画一个界面，AI 生成可运行的前端代码

# 提示词 ≠ 问问题

**Prompt Engineering（提示词工程）本质上是在写"工作说明书"，不是在"问问题"。**

把 AI 当人，你要给它的不是一个问题，而是一份完整的任务简报：

### 四个核心要素

| 要素 | 作用 | 例子 |
|------|------|------|
| **Role（角色设定）** | 告诉 AI 它是谁 | "你是有 10 年经验的后端系统架构师" |
| **Context（上下文）** | 给出背景信息 | "我们的服务运行在 K8s 上，单 Pod 限制 2GB 内存" |
| **Constraints（约束条件）** | 明确限制和要求 | "代码必须兼容 Go 1.18，不能用第三方库" |
| **Output Format（输出格式）** | 规定输出形式 | "用 Markdown 表格对比，最后给出推荐方案" |

<div class="callout">

**记住：** 你给 AI 的信息越详细，AI 给你的结果就越精准。
就像你给新同事布置任务 — 说得越清楚，活干得越好。

</div>

# System / User / Assistant

AI 对话的三个角色，用公司管理来理解：

### 三角色模型

| 角色 | AI 对话 | 公司类比 | 作用 |
|------|---------|---------|------|
| **System** | 系统提示词 | 公司制度和文化 | 设定 AI 的行为准则、身份、能力边界 |
| **User** | 用户消息 | 员工下达的具体任务 | 提供需求、上下文、约束条件 |
| **Assistant** | AI 回复 | 执行者的工作成果 | 根据制度和任务要求完成工作 |

### 实际 API 调用格式

```json
{
  "messages": [
    {"role": "system",    "content": "你是资深后端工程师，精通 Go/Python/分布式系统..."},
    {"role": "user",      "content": "用户登录流程超时，请分析可能的原因..."},
    {"role": "assistant", "content": "根据登录流程链路，超时可能由以下原因导致..."}
  ]
}
```

<div class="callout-blue">

**重点：** System 消息是最强大的控制手段。它决定了 AI 的"人格"和"能力范围"。
写好 System Prompt，等于给 AI 做了一次完整的岗前培训。

</div>

# 小结：从今天起，把 AI 当同事

### 三个认知升级

1. **AI 不是搜索引擎** — 它能理解上下文、进行推理、生成创造性内容
2. **提示词不是搜索关键词** — 它是工作说明书，越详细越好
3. **对话不是一次性的** — 它是持续协作，可以多轮迭代优化结果

### 从现在开始的行动指南

| 以前的做法 | 现在的做法 |
|-----------|-----------|
| "帮我写个函数" | "你是 XX 工程师，背景是 XX，需要实现 XX，约束是 XX" |
| 一句话问完就走 | 多轮对话，持续追问和优化 |
| 只用文字 | 文字 + 截图 + 代码 + 文档，多模态输入 |
| 照搬 AI 的输出 | 审查、测试、迭代，AI 是助手不是权威 |

<div class="callout">

**核心理念：把 AI 当一个刚入职的、聪明但不了解你业务的新同事。**
你的任务是：给清楚的背景、明确的要求、合理的约束。它会回报你高质量的成果。

</div>
