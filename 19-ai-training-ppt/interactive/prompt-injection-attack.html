提示词注入：AI 最大的安全威胁

看黑客如何劫持 AI Agent，以及如何防御

🔐 OpenClaw 的真实安全事件

高危 ClawJacked 漏洞
网站通过 WebSocket 劫持本地 Agent，零交互
攻击者在恶意网页中嵌入 JavaScript 代码，通过 WebSocket 连接到用户本地运行的 OpenClaw Agent。 由于 Agent 默认监听所有接口且无认证机制，攻击者可以在用户毫不知情的情况下， 完全控制 Agent 执行任意命令——包括读取文件、安装后门、窃取凭证。 用户只需访问一个恶意网页即可中招，无需任何点击或确认操作。
供应链 ClawHub 供应链攻击
1,467 个恶意 Skills，窃取 macOS 密码
安全研究人员在 ClawHub（OpenClaw 的官方插件市场）上发现了 1,467 个恶意 Skills。 这些 Skills 伪装成常用工具（如数据库客户端、API 测试工具），实际上会在安装时执行恶意代码， 通过 macOS 的 security 命令提取钥匙串中存储的密码和 API 密钥， 然后将数据发送到攻击者控制的服务器。部分恶意 Skills 的下载量已超过数千次。
CVE 7 个 CVE 漏洞
SSRF、认证绕过、路径遍历、远程代码执行
2024-2025 年间，OpenClaw 先后被披露 7 个严重 CVE 漏洞： 包括 SSRF（服务端请求伪造）允许攻击者访问内网资源； 认证绕过使未授权用户可以调用 Agent API； 路径遍历漏洞让攻击者读取服务器任意文件； 以及远程代码执行（RCE）漏洞，攻击者可通过精心构造的请求在服务器上执行任意代码。 这些漏洞影响了大量生产环境部署。
行业 Cisco 评价
"个人 AI Agent 是安全噩梦"
Cisco Talos 安全研究团队在 2025 年的报告中指出： "个人 AI Agent 正在成为企业安全的噩梦。它们拥有广泛的系统权限， 能够执行文件操作、网络请求、代码执行等高危操作， 但大多数 Agent 框架缺乏基本的安全防护——没有沙箱隔离、没有权限控制、没有审计日志。 结合提示词注入攻击，攻击者可以轻松劫持这些 Agent， 将它们变成攻击内网的跳板。"该报告引发了业界对 AI Agent 安全的广泛关注。
