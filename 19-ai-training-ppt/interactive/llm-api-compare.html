Google Gemini

gemini-pro 
curl "https://generativelanguage.googleapis.com/v1/models/gemini-pro:generateContent?key=$GOOGLE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "system_instruction": {
      "parts": [{"text": "你是资深后端工程师"}]
    },
    "contents": [{
      "parts": [{"text": "OAuth 2.0 的认证流程是什么？"}]
    }],
    "generationConfig": {
      "temperature": 0.3,
      "maxOutputTokens": 1024
    }
  }'

差异标注

认证方式 GPT: Authorization: Bearer token
Claude: x-api-key 自定义 Header
Gemini: URL 查询参数 ?key=
系统提示 GPT: 放在 messages 数组中，role="system"
Claude: 独立的顶层 "system" 字段
Gemini: 独立的 "system_instruction" 对象，内含 parts
消息结构 GPT: messages: [{role, content}]
Claude: messages: [{role, content}]（与 GPT 相似）
Gemini: contents: [{parts: [{text}]}]（嵌套更深）
Token 限制参数 GPT: max_tokens
Claude: max_tokens（必填）
Gemini: generationConfig.maxOutputTokens

Google Gemini

响应格式 
{
  "candidates": [{
    "content": {
      "role": "model",
      "parts": [{
        "text": "OAuth 2.0 的认证流程包括..."
      }]
    },
    "finishReason": "STOP",
    "safetyRatings": [...]
  }],
  "usageMetadata": {
    "promptTokenCount": 28,
    "candidatesTokenCount": 256,
    "totalTokenCount": 284
  }
}

差异标注

取文本路径 GPT: choices[0].message.content
Claude: content[0].text
Gemini: candidates[0].content.parts[0].text
结束原因 GPT: finish_reason: "stop"
Claude: stop_reason: "end_turn"
Gemini: finishReason: "STOP"（大写）
用量统计 GPT: usage.prompt_tokens / completion_tokens
Claude: usage.input_tokens / output_tokens
Gemini: usageMetadata.promptTokenCount / candidatesTokenCount
内容结构 GPT: content 是字符串
Claude: content 是数组（支持多块内容，如文本+工具调用）
Gemini: parts 数组（嵌套最深）

Google Gemini

工具定义 + 调用 
// 请求：定义工具
{
  "tools": [{
    "functionDeclarations": [{
      "name": "query_logs",
      "description": "查询系统日志",
      "parameters": {
        "type": "OBJECT",
        "properties": {
          "request_id": {
            "type": "STRING",
            "description": "请求ID"
          }
        },
        "required": ["request_id"]
      }
    }]
  }],
  "contents": [...]
}

// 响应：模型调用工具
{
  "candidates": [{
    "content": {
      "parts": [{
        "functionCall": {
          "name": "query_logs",
          "args": {
            "request_id": "REQ-001"
          }
        }
      }]
    },
    "finishReason": "STOP"
  }]
}

差异标注

工具定义 GPT: tools[].function.parameters（JSON Schema）
Claude: tools[].input_schema（JSON Schema）
Gemini: tools[].functionDeclarations[].parameters（类型大写：OBJECT, STRING）
调用格式 GPT: message.tool_calls[]，arguments 是 JSON 字符串
Claude: content[] 中 type="tool_use" 块，input 是对象
Gemini: parts[].functionCall，args 是对象
参数传递 GPT: arguments: "{\"request_id\":\"REQ-001\"}"（字符串，需 JSON.parse）
Claude: input: {"request_id": "REQ-001"}（已解析的对象）
Gemini: args: {"request_id": "REQ-001"}（已解析的对象）
并行特性 GPT: tool_calls 数组可含多个调用（原生并行）
Claude: content 数组可含文本+多个 tool_use（混合内容）
Gemini: parts 数组可含多个 functionCall

Google Gemini

streamGenerateContent 
// 使用 streamGenerateContent 端点
// 响应为 JSON 数组流:

[{
  "candidates": [{
    "content": {
      "parts": [{
        "text": "OAuth"
      }],
      "role": "model"
    }
  }]
},
{
  "candidates": [{
    "content": {
      "parts": [{
        "text": "2.0 的认证"
      }]
    }
  }]
},
{
  "candidates": [{
    "content": {
      "parts": [{
        "text": "流程包括..."
      }]
    },
    "finishReason": "STOP"
  }],
  "usageMetadata": {...}
}]

差异标注

协议格式 GPT: SSE（data: JSON），以 data: [DONE] 结束
Claude: SSE（event + data），有明确的生命周期事件
Gemini: JSON 数组流（非标准 SSE），直接返回 JSON 对象
增量字段 GPT: choices[0].delta.content
Claude: delta.text（在 content_block_delta 事件中）
Gemini: candidates[0].content.parts[0].text
事件粒度 GPT: 无事件类型，每个 chunk 结构相同
Claude: 细粒度事件：message_start → content_block_start → delta → stop
Gemini: 无事件类型，每个 chunk 是完整的 candidate 对象
启用方式 GPT: "stream": true 参数
Claude: "stream": true 参数
Gemini: 更换端点为 streamGenerateContent
