Host 的职责

Host 是 MCP 架构的最顶层，负责管理 AI 应用的生命周期。它可以创建多个 Client 实例，每个 Client 连接到不同的 Server。Host 还负责用户界面渲染和权限管理。

// Host 创建 Client 实例
const client = new MCPClient({
  name: "my-app",
  version: "1.0.0"
});

Client 的职责

每个 Client 维护与一个 Server 的连接。它负责协议握手、能力协商、消息序列化与反序列化。Client 是 Host 与 Server 之间的桥梁，确保通信符合 MCP 规范。

// Client 连接到 Server
await client.connect(
  new StdioTransport({
    command: "npx",
    args: ["@mcp/fs-server"]
  })
);

Server 的职责

Server 是功能的实际提供者。它向 Client 声明自己的能力（有哪些工具、资源、提示模板），并在收到调用请求时执行具体操作。每个 Server 专注于一个领域。

// Server 注册工具
server.setRequestHandler(
  ListToolsRequestSchema,
  async () => ({
    tools: [{
      name: "read_file",
      description: "读取文件内容"
    }]
  })
);

Server 对外暴露的三种标准化能力类型
🛠️
Tools
工具 - 可执行的操作
read_file run_query search_web
📁
Resources
资源 - 可读取的数据
file://src/*.ts db://users api://weather
📝
Prompts
提示模板 - 可复用的指令
code-review summarize translate

三种能力的区别

Tools 是模型主动调用的操作，会产生副作用（如写文件、发请求）。Resources 是只读的数据源，供应用程序读取上下文。Prompts 是预定义的指令模板，由用户选择触发。

实际通信示例
→ Client 发送 — 初始化请求
// 建立连接，声明客户端能力 { "jsonrpc": "2.0", "id": 1, "method": "initialize", "params": { "protocolVersion": "2024-11-05", "capabilities": {} } }
← Server 响应 — 返回能力列表
// Server 声明支持的功能 { "jsonrpc": "2.0", "id": 1, "result": { "protocolVersion": "2024-11-05", "capabilities": { "tools": {} } } }
→ Client 发送 — 列出工具
// 查询 Server 有哪些工具 { "jsonrpc": "2.0", "id": 2, "method": "tools/list" }
← Server 响应 — 工具定义
// 返回可用工具的详细描述 { "jsonrpc": "2.0", "id": 2, "result": { "tools": [{ "name": "read_file", "description": "读取文件内容", "inputSchema": { "type": "object", "properties": { "path": { "type": "string" } } } }] } }
→ Client 发送 — 调用工具
// AI 决定调用 read_file 工具 { "jsonrpc": "2.0", "id": 3, "method": "tools/call", "params": { "name": "read_file", "arguments": { "path": "/src/main.go" } } }
← Server 响应 — 工具结果
// 返回文件内容 { "jsonrpc": "2.0", "id": 3, "result": { "content": [{ "type": "text", "text": "package main\n\nimport..." }] } }
