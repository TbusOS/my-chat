# Cursor 高阶使用演示指南

> 现场演示用，按步骤操作

## 准备

1. 打开 Cursor IDE
2. 打开一个示例项目（建议用公司某个非敏感模块）
3. 确保 Cursor 已登录且 AI 功能可用

## Demo 1：Agent 模式（Cmd+I / Ctrl+I）

### 步骤
1. 按 `Cmd+I` 打开 Composer
2. 切换到 **Agent** 模式（左下角切换）
3. 输入任务：

> 重构这个文件中的请求处理函数：
> 1. 把大函数拆分成小函数，每个不超过 30 行
> 2. 添加错误处理
> 3. 所有数值计算使用 decimal 而不是 float
> 4. 保持接口不变

4. 观察 Cursor 如何：
   - 自动读取相关文件
   - 规划修改方案
   - 逐步执行修改
   - 跨文件协调变更

### 要点讲解
- Agent 模式 ≠ 简单补全，它会主动阅读项目结构
- 它能理解文件之间的依赖关系
- 给它的指令越具体，结果越好

## Demo 2：@ 引用系统

### 步骤

1. 在 Chat 面板中使用各种 @ 引用：

```
@file:src/order/handler.go 这个文件里的错误处理模式有什么问题？
```

```
@folder:src/order/ 整个订单模块的架构合理吗？给出改进建议。
```

```
@web:https://docs.anthropic.com/en/docs/tool-use
根据这个文档，帮我实现 tool use 的调用
```

```
@codebase 项目里所有用到 float64 做数值计算的地方在哪里？
```

### 要点讲解
- @file: 精确引用单个文件
- @folder: 引用整个目录
- @web: 引用在线文档（实时爬取）
- @codebase: 全项目搜索

## Demo 3：.cursorrules 配置

### 步骤

1. 在项目根目录创建 `.cursorrules` 文件：

```
你是一位资深后端工程师。

代码规范：
- Go 语言使用 gofmt 标准格式
- 错误处理必须用 fmt.Errorf 包装上下文
- 数值计算一律使用 decimal 类型
- 所有公共函数必须有注释
- 变量命名使用 camelCase
- 禁止使用 panic，统一用 error 返回

安全要求：
- 敏感数据（用户信息、密码）不得打日志
- SQL 查询必须用参数化查询
- API 密钥从环境变量读取
```

2. 然后在 Chat 中写代码，观察 AI 如何遵循这些规则
3. 对比：有 .cursorrules vs 没有 .cursorrules 的输出差异

### 要点讲解
- .cursorrules = 给 AI 定公司制度
- 项目级配置，团队共享
- 越具体越好，不要泛泛而谈

## Demo 4：多文件编辑

### 步骤

1. 在 Composer Agent 模式输入：

> 为订单模块添加单元测试：
> - 测试正常请求流程
> - 测试参数边界（空值、非法格式、超长字符串）
> - 测试网络超时场景
> - 使用 table-driven test 模式

2. 观察 Cursor 如何：
   - 创建新的测试文件
   - 引用原始代码理解接口
   - 生成完整的测试用例
   - 确保测试能通过编译

## 小贴士

- **Tab 补全只是入门**，真正的效率提升在 Agent 模式
- **提示词越像工作指令**，结果越好（回扣第一部分主题）
- **.cursorrules 是团队标准化的利器**
- **@引用让 AI 有上下文**，避免"驴唇不对马嘴"
