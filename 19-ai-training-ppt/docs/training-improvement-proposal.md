## 1. 核心痛点诊断

在仔细分析了目前的 PPT 和 HTML 动画代码后，发现主要存在以下几个体验和认知上的断层：

1.  **排版拥挤截断**：由于 Marp 的默认尺寸限制（1280x720）与全局 `50px` 的大内边距、`28px` 的基础字号相冲突。像第 84 页（以及含有大型表格的页面）会出现内容溢出、底部文字被无情截断的问题。
2.  **认知顺序错位（先深后浅）**：在第一部分（`01-mindset-shift.md`）中，在讲完大概念（AGI/ASI）后，直接跳入了硬核技术（Transformer 注意力机制、Embedding 向量化）。小白此时连"AI 在实际中能到底自己干嘛"还没建立概念，直接讲原理容易导致听众走神。
3.  **大模型与 Agent 的概念混淆**：~~培训材料需要在最早期，用最通俗的语言划清界限。~~ **「部分已解决」** — 第 5 章（`05-agents-ecosystem.md`）已有清晰区分：大模型 = 有脑子但没有手脚的顾问，Agent = LLM + Tools + Memory + Planning。01 章新增了"高位截瘫的天才"比喻页作为预告，建立初步印象。
4.  **动画演绎缺乏"场景感"**：~~最初的交互 HTML 只是静态图标的变化，缺乏"Show, don't tell"的场景化能力。~~ **「已解决」** — 已重写全部交互动画，当前共 11 个交互式 HTML，均支持单步/自动播放/键盘控制，详见下方第 4 节清单。

## 2. 排版与视觉优化建议（CSS）

为了解决文字溢出和在大屏幕/投影仪上的展示效果，建议直接修改 `theme/ai-training.css`：

*   **缩小全局留白与字号**：
    将全局的 padding 和字体缩小约 15%，给内容留出呼吸空间。
    ```css
    /* 建议修改 theme/ai-training.css 中的 section 样式 */
    section {
      padding: 40px 50px; /* 原为 50px 70px */
      font-size: 24px;    /* 原为 28px */
      line-height: 1.6;
    }
    ```
*   **长页面强制拆分**：
    如果在修改 CSS 后仍有页面（尤其是包含大量 Markdown 表格的页面）显示不全，**坚决拆页**。例如把《Agent 工作流全过程》拆为 `工作流（上半场：分析与规划）` 和 `工作流（下半场：执行与反馈）`。
*   **投影仪对比度适配**：
    将 CSS 中偏淡的灰色（如 `--color-text-light: #64748b`）加深为 `#475569`，确保投影在幕布上不泛白。

## 5. 给讲师的思维转化建议（Talk Track）

对于培训小白的技术主管来说，最忌讳的是"陷入专业术语的自嗨"。请在讲解时牢记以下原则：

1.  **戒掉 API 视角，拥抱"拟人"视角**。不要说"我们将请求发给大模型，它解析 JSON 返回结果"，而要说"你给这位新员工发了一封邮件，他看了以后采取了行动"。
2.  **用"权限"解释 Agent**。小白很难理解什么是 Tool Use / Function Calling。你只需要说："你平时不让你家狗上桌吃饭（没权限）。Agent 就是你给了 AI 一把打开你电脑各个软件的钥匙，允许它上桌。"
3.  **强调"闭环"**。现在的软件是"你按一个按钮，它出一个结果"；Agent 的核心是"你定一个目标，它如果遇到报错，会**自己想办法重试并修正**，直到完成目标"。
