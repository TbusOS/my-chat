# 单卡与低配运行指南

## 目标

没有 8 卡 H100 也能体验 nanochat 的完整流程。

## 方案一：单卡 NVIDIA GPU

nanochat 的所有代码在单卡上都能正常运行，区别只是速度。

### 原理

去掉 `torchrun` 后，代码自动切换为梯度累积模式，产出和多卡几乎相同的结果。

### 运行方式

```bash
# 单卡训练（不使用 torchrun）
python -m scripts.base_train -- --depth=12

# 单卡 SFT
python -m scripts.chat_sft
```

### VRAM 不够 80GB？

调整 `--device_batch_size`：

```bash
# 默认 32，逐步减小直到不 OOM
python -m scripts.base_train -- --depth=12 --device_batch_size=8
```

| VRAM | 建议 batch size | 说明 |
|------|-----------------|------|
| 80GB | 32（默认） | 正常运行 |
| 48GB | 8-16 | RTX A6000 / 4090 |
| 24GB | 2-4 | RTX 3090 / 4090 |
| 16GB | 1-2 | 可能需要更小的 depth |

## 方案二：CPU / Apple Silicon

nanochat 提供了专门的 CPU/MPS 运行脚本：

```bash
bash runs/runcpu.sh
```

### 注意事项

- 这个脚本会大幅缩小模型规模以适应 CPU/MPS
- 训练时间在十几分钟左右
- **结果不会很好**，但足以理解完整流程
- Apple Silicon 上默认使用 float32（MPS 暂不支持 bf16 tensor core）

## 方案三：Google Colab

你也可以用 Colab 的免费 T4 GPU 运行，但需要注意：

- T4 的 VRAM 只有 16GB，需要很小的 depth（如 4-8）
- T4 不支持 bf16，需要设置环境变量：

```bash
export NANOCHAT_DTYPE=float16
```

## 精度说明

nanochat 不使用 `torch.amp.autocast`，而是通过 `COMPUTE_DTYPE` 显式控制：

| 硬件 | 默认精度 | 说明 |
|------|---------|------|
| CUDA SM 80+（A100、H100） | bfloat16 | 原生 bf16 tensor core |
| CUDA SM < 80（V100、T4） | float32 | 可手动设 `NANOCHAT_DTYPE=float16` |
| CPU / MPS | float32 | 无低精度 tensor core |

```bash
# 手动指定精度
NANOCHAT_DTYPE=float32 python -m scripts.chat_cli -p "hello"
NANOCHAT_DTYPE=bfloat16 torchrun --nproc_per_node=8 -m scripts.base_train
```

## 实际体验建议

| 你有什么 | 建议做法 |
|---------|---------|
| 8×H100/A100 | 直接跑 `speedrun.sh`，完整体验 |
| 单卡 3090/4090 | 跑 depth=12 的小模型，感受流程 |
| Mac M1/M2/M3 | 跑 `runcpu.sh`，理解 pipeline |
| 只有 CPU | 跑 `runcpu.sh`，重在理解不在结果 |

> 核心思路：`--depth` 是唯一的复杂度旋钮。小 depth = 小模型 = 少资源 = 结果差但流程一样。
