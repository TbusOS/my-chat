# nanochat 快速入门

## 目标

启动一台 GPU 机器，运行一个脚本，2-4 小时后在网页上和你自己训练的"迷你 ChatGPT"对话。

## 环境要求

| 项目 | 要求 |
|------|------|
| GPU | 8×H100（推荐）/ 8×A100 / 单卡也可以 |
| VRAM | 每卡 80GB（低于 80GB 需要调参） |
| Python | 3.10+ |
| 包管理 | 推荐 [uv](https://docs.astral.sh/uv/) |

### 推荐的 GPU 云服务

- [Lambda](https://lambda.ai/service/gpu-cloud) — Karpathy 本人使用
- [Vast.ai](https://vast.ai/) — Spot 实例更便宜
- [RunPod](https://www.runpod.io/)

## 步骤

### 1. 克隆项目

```bash
git clone https://github.com/karpathy/nanochat.git
cd nanochat
```

### 2. 安装依赖

```bash
# 推荐用 uv（nanochat 项目自带 pyproject.toml）
uv sync

# 或用 pip
pip install -e .
```

### 3. 一键训练

```bash
# 在 screen 或 tmux 里运行（需要 2-4 小时）
bash runs/speedrun.sh
```

这个脚本会自动完成：
1. 训练 BPE 分词器
2. 预训练基座模型（depth=20~26）
3. SFT 监督微调
4. 评估模型能力

### 4. 启动 Web 聊天

训练完成后：

```bash
source .venv/bin/activate
python -m scripts.chat_web
```

然后在浏览器访问显示的 URL。如果是云服务器，用公网 IP + 端口，例如 `http://209.20.xxx.xxx:8000/`。

### 预期效果

- 你可以像使用 ChatGPT 一样和它对话
- 让它写故事、写诗
- 问它天空为什么是蓝色的
- 问它"你是谁"会看到有趣的幻觉
- 它大概相当于一个幼儿园水平的 AI（毕竟只花了 $48）

## 常见问题

### Q: 运行 speedrun.sh 报 OOM？

减小 batch size。找到脚本中的 `--device_batch_size` 参数，从 32 逐步减小到 16、8、4、2、1。

### Q: 训练中断了怎么办？

nanochat 会自动保存 checkpoint，重新运行脚本会从上次中断处继续。
