# 从预训练到对话的完整流程

## 概述

nanochat 的核心价值是把训练一个 ChatGPT 的**完整流程**压缩到一个可跑通的代码库里。这个流程和 OpenAI 训练 ChatGPT 的步骤本质上是一样的：

```
分词器训练 → 预训练 → SFT 微调 → 强化学习 → 评估 → 部署对话
```

## 阶段 1：分词器训练 (tok_train.py)

**做什么**：训练一个 BPE 分词器，把文本转换为 token 序列。

**原理**：和本仓库 [7-tokenizer](../../7-tokenizer/) 讲的 BPE 算法一致，只是 nanochat 的分词器更接近 GPT-4 的实现风格。

```bash
python -m scripts.tok_train
```

**输出**：分词器模型文件，后续所有阶段都会用到。

## 阶段 2：预训练 (base_train.py)

**做什么**：在大规模文本数据上训练基座模型，让它学会"语言"。

**原理**：
- 标准的 next-token prediction
- 训练数据：FineWeb / NVIDIA ClimbMix 等大规模语料
- 支持 torchrun 多卡分布式训练
- 自动梯度累积适配不同卡数

```bash
# 8 卡分布式
OMP_NUM_THREADS=1 torchrun --standalone --nproc_per_node=8 -m scripts.base_train -- --depth=26

# 单卡
python -m scripts.base_train -- --depth=12
```

**这是最耗时的阶段**，GPT-2 级别（depth=26）在 8×H100 上约需 2 小时。

### 监控指标

| 指标 | 含义 |
|------|------|
| val_bpb | 验证集 bits per byte（越低越好） |
| core_metric | DCLM CORE 评分 |
| train/mfu | 模型 FLOPS 利用率 |
| train/tok_per_sec | 训练吞吐量 |

## 阶段 3：基座评估 (base_eval.py)

**做什么**：评估预训练后的基座模型质量。

```bash
python -m scripts.base_eval
```

**评估内容**：
- DCLM CORE score（GPT-2 基准为 0.2565）
- Bits per byte
- 文本生成样本

## 阶段 4：SFT 监督微调 (chat_sft.py)

**做什么**：教基座模型"如何对话"。

**原理**：
- 基座模型只会续写文本，不会对话
- SFT 用对话数据（如 HuggingFace SmolTalk）微调模型
- 微调后模型学会了"问答"格式

```bash
python -m scripts.chat_sft
```

**和本仓库的关联**：这一步本质上就是 [2-litgpt-finetune](../../2-litgpt-finetune/) 讲的微调，只是数据换成了对话格式。

## 阶段 5：强化学习 (chat_rl.py)

**做什么**：进一步优化对话质量（可选）。

**原理**：
- 和本仓库 [9-rlhf-alignment](../../9-rlhf-alignment/) 讲的对齐技术相关
- 通过奖励信号让模型学会更好的回答

```bash
python -m scripts.chat_rl
```

## 阶段 6：对话评估 (chat_eval.py)

**做什么**：评估微调后模型的对话能力。

```bash
python -m scripts.chat_eval
```

**评估任务**：
- GSM8K — 数学推理
- MMLU — 多领域知识
- HumanEval — 代码生成
- ARC — 科学推理
- SpellingBee — 拼写/字母计数

## 阶段 7：部署对话

### Web UI（推荐）

```bash
python -m scripts.chat_web
```

FastAPI 服务同时提供 Web 前端和 API，在浏览器中使用类似 ChatGPT 的界面。

### CLI

```bash
python -m scripts.chat_cli -p "你好，请介绍一下你自己"
```
